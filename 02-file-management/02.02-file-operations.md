# 02.02 File Operations

## Purpose

Create, copy, move, inspect, and remove files.

This page handles direct state changes, so target verification matters.

## Strict Syntax

```bash
touch <file>
cp <source> <target>
mv <source> <target>
rm <file>
rm -r <directory>
stat <file>
file <file>
```

## Command Map

| Command | Use |
|---|---|
| `groups` | Show current user group membership. |
| `stat` | Show inode, ownership, mode, and timestamps. |
| `file` | Identify file payload type. |
| `touch` | Create file or update timestamp. |
| `cp` | Copy file data. |
| `mv` | Move or rename a file. |
| `rm` | Remove a file. |
| `rm -r` | Remove a directory tree. |

## Atomic Examples

```bash
groups
stat app.conf                         # show inode, ownership, mode, and timestamps
file app.conf                         # identify content type, not extension
ls -l app.conf
touch notes.txt
touch --no-create existing.txt        # update timestamp only if the file exists
touch -t 202605271030.00 report.txt   # set timestamp to YYYYMMDDHHMM.SS
touch --reference=source.txt target.txt # copy timestamp from source to target
cp source.txt target.txt
cp -f source.txt target.txt           # overwrite target if needed
cp -p source.txt target.txt           # preserve mode, ownership, and timestamps
cp --parents etc/app.conf backup/     # recreate source parent path under backup
cp --target-directory=backup file1 file2 # send multiple sources to one directory
mv old.txt new.txt
mv report.txt archive/
rm old.txt
rm -r build/                          # remove directory tree recursively
rm -ri build/                         # recursive removal with prompts
```

## Glob and Brace Examples

```bash
cp *.sh backup/
ls file?.txt                          # ? matches exactly one character
rm [A-Z]*.tmp                         # target names starting with uppercase letters
rm [!a-z]*                            # target names not starting with lowercase letters
printf '%s\n' file{1..3}.txt          # brace expansion generates file1..file3
printf '%s\n' {fast,slow}-server.conf # brace expansion generates two arguments
```

## Compound Scenarios

Copy a config while preserving metadata:

```bash
stat app.conf
cp -p app.conf app.conf.bak           # keep original metadata on backup
stat app.conf.bak
```

Move logs into a dated archive directory:

```bash
mkdir -p archive/2026-05-27
mv *.log archive/2026-05-27/
ls -l archive/2026-05-27/
```

Review before removal:

```bash
pwd
ls -l *.tmp
rm -i *.tmp
```

## Exam Traps

`touch` updates timestamps when a file already exists.

`cp -f` can overwrite existing content.

`mv` renames when source and target are in the same directory.

Globs are expanded by the shell before the command runs.

Brace expansion is text generation, not filesystem matching.

## Production Notes

Avoid blind `rm -rf`.

Use `rm -i` or a dry listing when deleting broad targets.

Use `stat` before and after metadata-sensitive operations.
