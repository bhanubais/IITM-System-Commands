# 01.02 WSL Initialization

## Purpose

Create and maintain a Linux environment through Windows Subsystem for Linux.

Hardware virtualization must be enabled before this layer works correctly.

## Strict Syntax

```powershell
wsl -l -v
wsl -l -o
wsl --install
wsl --install -d <DistroName>
wsl --update
```

## Command Map

| Command | Use |
|---|---|
| `wsl -l -v` | List installed distributions and WSL versions. |
| `wsl -l -o` | List distributions available for install. |
| `wsl --install` | Install the default Linux distribution. |
| `wsl --install -d` | Install a specific distribution. |
| `wsl --update` | Update the WSL kernel. |

## Atomic Examples

```powershell
wsl -l -v
wsl -l -o
wsl --install
wsl --install -d Ubuntu
wsl --update
```

## Compound Scenarios

Inspect state before installing:

```powershell
wsl -l -v
wsl -l -o
wsl --install -d Ubuntu
```

Refresh WSL before course work:

```powershell
wsl --update
wsl -l -v
```

Confirm distribution version drift:

```powershell
wsl -l -v
wsl --update
wsl -l -v
```

## Exam Traps

`wsl -l -v` lists installed distributions.

`wsl -l -o` lists installable distributions.

`wsl --update` updates WSL itself, not packages inside the Linux distribution.

## Production Notes

Verify the distribution state before debugging Linux commands on Windows.

Keep WSL updated when kernel behavior matters.

Treat WSL distribution choice as part of the lab baseline.
