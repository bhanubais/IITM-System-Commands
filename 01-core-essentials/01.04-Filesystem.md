## Linux has four major parts

The four major parts of Linux, conceptually. **Low-level kernel functions** are called by **programs**, which **are invoked in a shell**, which can be **launched by a graphical desktop**.

1. **Kernel**: Connects hardware, networks etc.
2. **Programs**: Call kernel. Programs those run on Terminal, called `commands`
3. **Shell**: Invoke programs in CLI such as `bash`, `dash`, `ksh`, etc.
4. **Graphic Desktop Environment** (Optional): Launch Shell. UI with windows such as GNOME, KDE, etc.

![Four major parts of the Linux](imgs/linux_major_parts.png)


## Shell Operation

1. **Read**: Input from file/terminal, arguments
2. **Break** into `tokens`: Words and operators separated by `metacharacters`
3. **Parse**: `tokens` into commands
4. **Shell expansions** (opt.):
Break expanded tokens into lists of filenames, commands, and arguments.
5. **Redirection** (opt.):
Perform redirection and removes operators and their operands from argument list.
6. **Execute**:
7. **Waits** (opt.): Waits for command to complete and collects its exit status.


## System Directories

![Directory Scope](imgs/scope.png)

### Scope:
- `/`: Root (System files)
- `/usr`: User (More System files, supplied by Distro)
- `/usr/local`: User Local (Added System files)

### Category:

**Programs**:
- `bin`: Locally installed programs
- `sbin`: programs for superusers.
- `lib`: library files for programming.

**Documentations**:
- `doc`: Documentation
- `info`: Docs for emacs's built-in help system.
- `man`: manual pages
- `share`: program-specific (examples and instructions)

**Configurations**:
- `etc`: configurations
- `init.d`: Linux booting configurations
- `rc.d`: More booting configs.; also `rc1.d`, `rc2.d`, ...

**Programming**:
- `include`: Header files for programming
- `src`: Source code for programs

**Web files**:
- `cgi-bin`: Scripts/programs that run on web pages.
- `html`: Web pages
- `public_html`: Web pages
- `www`: Web pages

**Display**:
- `fonts`: Installed Fonts
- `X11`: X window system files

**Hardware**:
- `dev`: Hardware Device files (drivers)
- `media`: Mount points
- `mnt`: Mount points

**Running files**:
- `var`: Frequently updated Stat files.
- `lock`: Lock files created by a program to prevent run same task.
- `log`: Logs for events, errors, warnings, and other information.
- `mail`: Mailboxes for incoming email
- `run`: PID files containing IDs of running processes to track or kill a particular process.
- `spool`: Queued or in transit such as email, print / schedule jobs.
- `tmp`: Temporary storage for programs and/or people to use

## Kernel-Related Directories

- `/boot`: Booting system files directly deals with kernel.
- `/lost+found`: Damaged files rescued by a disk recovery tool.
- `/proc`: Curring running process files.
- `/sys`: Kernel's internal files.

Files in `/proc` always appear to be **zero-sized**, **read-only**, and **dated now**, but their contents magically contain information about the Linux kernel.

Files in `/sys` also have misleading sizes and magical contents.

`/proc`, and `/sys` are used mostly by system programs, but feel free to view them. Few examples are below:

- `/proc/ioports`: List of input/output hardware.
- `/proc/cpuinfo`: Information about PC Processors.
- `/proc/version`: OS version
- `/proc/uptime` : Seconds elapsed since the system booted. Equivalent to `uptime` cmd.
- `/proc/NNN`    : **Folder**. Information about process id NNN. i.e. `/proc/108`.
- `/proc/self`   : **Folder**. Current process information.
