# 05.08 Paste, Tee, Xargs, and Delimited Data

## Purpose

Join streams.

Duplicate streams.

Convert pipeline output into command arguments.

Parse simple delimited records.

## Strict Syntax

```bash
paste file1 file2
paste -d DELIM file1 file2
paste -s -d DELIM file
command | tee file
command | tee -a file
producer | xargs command
while IFS=, read -r a b rest; do ...; done < file.csv
```

## Command Map

| Command | Purpose |
|---|---|
| `paste` | Join lines from files column-wise. |
| `paste -d` | Set output delimiter. |
| `paste -s` | Join one file serially. |
| `tee` | Write stream to file and stdout. |
| `tee -a` | Append instead of overwrite. |
| `xargs` | Build command arguments from stdin. |
| `IFS= read -r` | Read delimited fields safely. |

## Atomic Examples

```bash
paste names.txt scores.txt
paste -d ',' names.txt scores.txt
paste -s -d ' ' names.txt
printf '%s\n' a b c | tee letters.txt
printf '%s\n' a b c | tee -a letters.txt
find . -type f | xargs file
find . -type f -print0 | xargs -0 file
while IFS=, read -r name role rest; do printf '%s %s\n' "$name" "$role"; done < users.csv
```

## Compound Scenarios

Save an intermediate report while continuing the pipeline:

```bash
grep -i 'error' app.log | tee errors.raw | sort | uniq -c | sort -nr > errors.rank
```

Join two extracted columns:

```bash
cut -f1 users.tsv > names.tmp
cut -f3 users.tsv > shells.tmp
paste names.tmp shells.tmp
```

Classify files found by search:

```bash
find . -type f -name '*.sh' -print0 | xargs -0 file | sort
```

## Exam Traps

Plain `xargs` breaks on spaces and newlines inside filenames.

Use `find -print0 | xargs -0` for filename safety.

`tee` overwrites by default.

Simple `IFS=, read` is not a full CSV parser when quoted commas exist.

## Production Notes

Use `tee` when you need logs and a continued pipeline.

Use `paste` for quick two-column reports.

Use null-delimited `xargs` when processing filesystem names.
