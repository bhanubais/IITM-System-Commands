# 01.03 Hardware Audit and Reconnaissance

## Purpose

Identify the host, kernel, CPU, memory, disks, sessions, and hardware surfaces.

Use these commands before package, network, or performance diagnosis.

## Strict Syntax

```bash
uname -a
hostnamectl
who
last -n <count>
lscpu
free -h
df -h
lsblk
sudo lshw -short
sudo dmidecode --type memory
```

## Command Map

| Command | Use |
|---|---|
| `uname -a` | Kernel and machine summary. |
| `hostnamectl` | Host identity and OS metadata. |
| `hostname -f` | Fully qualified domain name. |
| `hostname -I` | Host IP addresses. |
| `lsb_release -a` | Distribution metadata. |
| `who` | Active login sessions. |
| `last` | Historical login sessions. |
| `lscpu` | CPU summary. |
| `free -h` | Memory and swap state. |
| `df -h` | Filesystem capacity. |
| `lsblk` | Block devices. |
| `lshw` | Hardware tree. |
| `dmidecode` | SMBIOS and DMI records. |

## Atomic Examples

```bash
uname -a
cat /proc/version
hostnamectl
hostname -f
hostname -I
lsb_release -a
who
who --all --heading
last -n 5
ls -l /proc/self
lscpu
cat /proc/cpuinfo
cat /proc/meminfo
free -h
df -h
cat /proc/partitions
lsblk
sudo fdisk -l
sudo lshw -short
sudo lshw -c processor
sudo dmidecode --type memory
```

## Compound Scenarios

Build a quick incident host header:

```bash
printf 'host=%s\n' "$(hostname -f)"
uname -a
hostnamectl
```

Check memory and disk pressure:

```bash
free -h
df -h
lsblk
```

Extract CPU model for a report:

```bash
lscpu | grep -i 'model name' | cut -d: -f2 | sed 's/^ *//'
# 13th Gen Intel(R) Core(TM) i7-13620H
```

## Exam Traps

`/proc` and `/sys` expose virtual kernel state. Zero-byte listings do not mean empty operational data.

`hostname -i` and `hostname -I` can differ by distribution and name resolution behavior.

`lshw` and `dmidecode` need elevated privileges for complete data.

## Production Notes

Capture host identity before remote changes.

Check disk and memory pressure before blaming applications.

Prefer `lscpu`, `free`, `df`, and `lsblk` for fast triage.
