# 01.05 System Time and Scheduling

## Purpose

Inspect time.

Read calendars.

Schedule or cancel host power transitions.

## Strict Syntax

```bash
date
date -R
cal [month] [year]
ncal
shutdown -h now
shutdown --reboot +<minutes>
shutdown -c
```

## Command Map

| Command | Use |
|---|---|
| `date` | Print system date and time. |
| `date -R` | Print RFC 5322 formatted time. |
| `cal` | Show calendar. |
| `ncal` | Show alternate calendar layout. |
| `shutdown -h` | Halt the system. |
| `shutdown --reboot` | Reboot the system. |
| `shutdown -c` | Cancel pending shutdown. |

**Note**: to see the difference of `data -R` your distro must set the local timezone. If it is not set by default, you may set it using below commands:

```bash
# see what is current timezone
timedatectl
#                Local time: Mon 2026-06-01 12:19:06 UTC
#            Universal time: Mon 2026-06-01 12:19:06 UTC
#                  RTC time: Mon 2026-06-01 12:19:07
#                 Time zone: Etc/UTC (UTC, +0000)
# System clock synchronized: yes
#               NTP service: active
#           RTC in local TZ: no

# First find your timezone
timedatectl list-timezones | grep Kolkata
# Asia/Kolkata

# Set your timezone
sudo timedatectl set-timezone Asia/Kolkata

# Confirm by either date command or timedatectl command
date -R
# Mon, 01 Jun 2026 17:57:39 +0530

```

## Atomic Examples

```bash
date
date -R
cal
cal apr 2026
ncal
shutdown -h now
shutdown --reboot now # or
reboot
shutdown --reboot +5
shutdown -h 13:00
shutdown -c
```

## Compound Scenarios

Record a timestamped maintenance note:

```bash
printf 'maintenance_start=%s\n' "$(date -R)" >> maintenance.log
```

Schedule a short reboot window:

```bash
shutdown --reboot +5
shutdown -c
```

Check the target month before planning:

```bash
date
cal apr 2026
ncal
```

## Exam Traps

`shutdown -h now` halts.

`shutdown --reboot now` reboots.

`shutdown -c` cancels a pending shutdown.

## Production Notes

Time matters for logs, TLS certificates, cron behavior, and distributed systems.

Record maintenance timestamps in a consistent format.

Use scheduled reboot commands carefully on shared systems.
