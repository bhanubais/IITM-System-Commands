# 03.03 Archival and Compression Protocols

Control static data aggregation and compress binary footprints locally prior to network transmission layers or cold storage.

## Real World Scenario: Cloud Codebase Backup
When administering live servers, modifying core data blocks without a rollback state causes fatal downtime. Before initiating upgrades in `/var/www/html`, administrators back up the application layer entirely.

1. Package and compress the live directory:
   `tar -czf site_backup_01.tar.gz /var/www/html`
2. Validate the capacity of the backup block:
   `du -h site_backup_01.tar.gz`
3. If an upgrade breaks the site, delete the corrupted directory and extract the payload securely:
   `tar -xzf site_backup_01.tar.gz -C /`

## Tape Archival Structure (tar)
Aggregate localized data trees into singular deployable payloads (`.tar`).
* `tar -cf archive.tar <dir>` : Compile the designated directory strictly into an uncompressed payload map.
* `tar -czf archive.tar.gz <dir>` : Compile and execute GZIP compression algorithms synchronously. Highly optimal for web server log rotation.
* `tar -xf archive.tar` : Extract the complete internal tree out into the current working filesystem.
* `tar -xzf archive.tar.gz` : Extract and decompress securely without intermediate manual file handling.
* `tar -tf archive.tar.gz` : List the internal target tree silently without extracting data to the disk. Excellent for diagnosing payload structures securely.

## Standard Compression Algorithms
Linux relies upon localized stream compressors wrapping individual files structurally (`.gz`, `.xz`, or `.bz2`).
* `gzip <file>` : Compress the specified node, permanently removing the original uncompressed source file.
* `gunzip <file.gz>` : Decompress the GZIP payload and remove the `.gz` archive file.
* `xz <file>` : Deploy high-density compression algorithms executing heavier CPU calculations. Produces smaller footprints ideal for limited bandwidth nodes.
* `bzip2 <file>` : Alternate compression specification yielding dense block storage constraints.

## Universal Archives
Many Windows and multi-OS environments lack native TAR capabilities. Exporting structures via ZIP maintains compatibility.
* `zip archive.zip <files>` : Deploy standard compression protocols ubiquitous across multi-OS networks.
* `unzip archive.zip` : Revert data out of generalized ZIP packages explicitly.
