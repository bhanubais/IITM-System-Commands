# 05.01 Stream Viewing and Log Tailing

## Purpose

Inspect file contents without editing them.

Use these commands for fast log viewing, paging, and line counts.

## Strict Syntax

```bash
cat <file>
less <file>
more <file>
head -n <lines> <file>
tail -n <lines> <file>
tail -f <file>
wc <file>
wc -l <file>
```

## Command Map

| Command | Use |
|---|---|
| `cat` | Print full file contents. |
| `less` | Page through a file with forward and backward movement. |
| `more` | Legacy page viewer. |
| `head -n` | Print first lines. |
| `tail -n` | Print last lines. |
| `tail -f` | Follow appended lines live. |
| `wc` | Count lines, words, and bytes. |
| `wc -l` | Count lines only. |

## Atomic Examples

```bash
cat README.md
cat file1 file2
less /var/log/syslog
more README.md
head -n 10 access.log
tail -n 20 access.log
tail -f /var/log/apache2/access.log
wc README.md
wc -l access.log
```

## Compound Scenarios

Watch active web requests:

```bash
tail -n 20 /var/log/apache2/access.log
tail -f /var/log/apache2/access.log
```

Inspect a large log safely:

```bash
less /var/log/syslog
```

Count records after filtering:

```bash
grep -i 'failed' /var/log/auth.log | wc -l
```

## Exam Traps

`cat` dumps the whole file to stdout.

`less` is safer than `cat` for large files.

`tail -f` keeps running until interrupted.

`wc` output order is lines, words, bytes.

## Production Notes

Use pagers for large logs.

Use `tail -f` for live incident observation.

Use `wc -l` for record counts.
