# 02.05 Diagnostics and Search

Execute deep traversal operations and diagnostic telemetry across filesystem hardware blocks.

## Search Operations
* `find <query> -type f` : Extract exact file node matches from recursive directory parsing.
* `find <query> -size +1G` : Isolate matches strictly exceeding specific binary sizes.
* `find <query> -mtime -7` : Locate nodes possessing modification timestamps within the preceding 7 days.
* `xargs` : Convert Standard Input pipelines into active parameter strings for target commands. Example: `find -type f -name "*.tmp" | xargs rm` bridges isolated queries into mass execution.

## Storage Diagnostics
* `df -h` : Report aggregate mount-point allocations and capacity utilization limits formatted rationally (MB/GB).
* `du -sh <dir_name>` : Output the summarized block utilization strictly consumed by the specified directory node.

## File Analytics
* `file <node>` : Identify file structural contents and payload origins regardless of visual extension bindings.
* `stat <node>` : Output exhaustive mapping of internal node Inode parameters, including precise ownership IDs, timestamp limits, and device mount links.
