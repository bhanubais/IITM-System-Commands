# 02.05 Diagnostics and Search

## Purpose

Search filesystem trees.

Measure storage usage.

Classify file payloads and metadata.

## Strict Syntax

```bash
find <path> <predicate> <action>
find . -type f -name '<pattern>'
find . -type f -size +<size>
find . -type f -mtime -<days>
df -h
du -sh <path>
file <path>
stat <path>
```

## Command Map

| Command | Use |
|---|---|
| `find` | Recursive search. |
| `find -type f` | Match files. |
| `find -type d` | Match directories. |
| `find -name` | Match basename pattern. |
| `find -size` | Match by size. |
| `find -mtime` | Match by modification time. |
| `df -h` | Show filesystem capacity. |
| `du -sh` | Summarize path usage. |
| `file` | Classify content. |
| `stat` | Show inode and metadata. |

## Atomic Examples

```bash
find . -type f
find . -type d
find . -type f -name '*.log'          # quote pattern so shell does not expand it
find . -type f -size +1G              # files larger than 1 GiB
find . -type f -mtime -7              # modified within the last 7 days
find . -type f -empty -print          # preview empty files
find . -type f -empty -print -delete  # print then delete matches
df -h
du -sh /var/log                       # summarize one directory
file script.sh                        # classify content
stat script.sh                        # show inode and metadata
ls -ld /*                             # inspect root children themselves
ls -ld /* | grep '^l'                 # filter symbolic links
ls -la | grep '^d'                    # filter directories
ls -la | grep '^[^d]'                 # filter non-directories
```

## Compound Scenarios

Find large files during disk pressure:

```bash
df -h
find /var -type f -size +1G -print
du -sh /var/log
```

Remove empty files after a print check:

```bash
find . -type f -empty -print
find . -type f -empty -print -delete
```

Classify shell scripts safely:

```bash
find . -type f -name '*.sh' -print0 | xargs -0 file
```

## Exam Traps

Quote `find -name` patterns so the shell does not expand them first.

`-mtime -7` means modified within the last seven days.

Test destructive `find` actions with `-print` first.

Use `-print0` with `xargs -0` for filenames containing spaces or special characters.

## Production Notes

Start disk incidents with `df -h`, then isolate with `du` and `find`.

Use `stat` when permissions, ownership, timestamps, or inodes matter.

Use `file` when extensions are unreliable.
