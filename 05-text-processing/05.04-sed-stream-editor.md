# 05.04 Sed Stream Editor

`sed` functions as an automated language execution processor for input text blocks. Operations compile line by line within internal active-pattern memory structures.

## Fundamental Operations
The primary execution format follows `:label address_pattern action options;`

Scale logic operations using grouping execution arrays: `{ cmd; cmd; }`

### Structural Action Commands
* `p` : Force output the contents of the pattern buffer space.
* `d` : Purge the complete pattern buffer block from memory.
* `s` : Drive pattern-replacement logic syntax (`s/query/replacement/`). Target global substitution by appending the `g` parameter.
* `=` : Export the structural integer line number natively.
* `w <filename>` : Transmit memory payload directly to a secondary target file.

## Text Pipeline Applications
Execute stream mutations directly at the command prompt. Let `-e` designate explicit inline scripts. Let `-E` implement Extended Regex parameters.

* `sed -e 's/old/new/g' <filename>` : Globally execute term replacement within the data stream.
* `sed -e '3,$d' <filename>` : Audit the file and delete the array spanning from integer line 3 to sequence termination `$`.
* `sed -e '1~3c ------censored--------' <filename>` : Sequentially execute modifications beginning line 1 and altering every third structural iteration using a static overlay block.
* `sed -e '/^#/d' -e '/^$/d' <filename>` : Scale internal pipeline operations to eradicate comment structures (`^#`) and empty breaks (`^$`).

## Advanced Execution Flow
Deploy operational `.sed` scripts using `sed -f <script_name> <filename>`.

Engine execution allows direct structural conditionals and loops. Apply the logical branch array (`b label`) or execution block test conditions (`t label` bounding successful mutations vs `T label` handling logical failures). Join isolated execution blocks structurally traversing new lines via `N` and line-replacement parsing arrays.
