## Restart/Shutdown the OS:

```bash
# Restart linux now
❯ sudo reboot # or
❯ sudo shutdown --reboot now

# shuts down the PC gracefully
# (stop services, save data, unmount drives, and turn off PC)
❯ sudo shutdown now
# ↑ The Best Choice. It alerts users, stops services cleanly, and powers off.
❯ sudo shutdown -h now
❯ sudo reboot --poweroff

# Schedule a shutdown i.e. at 3:15 pm (User 24h clock)
❯ sudo shutdown -h 15:15

# Schedule a reboot i.e. at 3:15 pm (User 24h clock)
❯ sudo shutdown --reboot -h 15:15

# Reboot in 10 minutes
❯ sudo shutdown --reboot +10 &

# Cancel all pending shutdown/reboot operation
❯ sudo shutdown -c

# Emergency but Dangerous: Instance restart
# Running this can cause data loss or corrupt your filesystem. Only use it as a last resort before pulling the physical power plug.
❯ sudo reboot --force
```

## Reset an unresponsive Shell

Sometimes a terminal is not displaying the keystrokes you type because the killed program could not close itself properly. To fix the shell, follow these steps:

1. Press `Ctrl + j` to get a shell prompt. This keystroke may work even if pressing Enter does not.
2. Type the command `reset` (even if the letters don't appear while you type) and press `ctrl + j` to run it. The shell should return to normal.


## The Magic Sequence: REISUB

The Magic SysRq key combination is a safety mechanism built directly into the Linux kernel. It allows you to communicate with the core of the operating system even if your desktop, mouse, and keyboard seem completely frozen.
By pressing a specific sequence of keys, you can safely close files, unmount drives, and restart the PC without corrupting your data.

To safely restart a frozen Linux machine, hold down the Alt and SysRq (usually the Print Screen key) keys together, and then type the following letters slowly, waiting about 2–3 seconds between each press:
R ➔ E ➔ I ➔ S ➔ U ➔ B
A helpful way to remember this sequence is the phrase: "Reboot Even If System Utterly Broken".

### What Each Key Does

- **R** (Raw): Takes keyboard control away from the frozen desktop interface and gives it back to the core kernel.
- **E** (tErminate): Sends a gentle signal (SIGTERM) to all running programs, asking them to save data and close.
- **I** (kIll): Force-closes (SIGKILL) any stubborn programs that refused to close during the previous step.
- **S** (Sync): Flushes all cached data from the RAM down into your hard drive to prevent data loss.
- **U** (Unmount): Remounts all hard drives as "Read-Only" so no new data can accidentally corrupt them.
- **B** (ReBoot): Instantly restarts the computer safely.

### How to Check if It Is Enabled

Many modern Linux systems disable some of these keys by default for security reasons. You can check if your system allows them by running:

```bash
❯ cat /proc/sys/kernel/sysrq
```


- If the output is 1, it is completely enabled and ready to use.
- If the output is 16, 176, or any other number, it is partially restricted.

**Note**: It can't be enable in the wsl based Linux as it lives under another os (Windows). If you have a standalone Linux PC then either it is enable by default or you may enable it. (To enable it, search on the internet, We have many other things to cover. 😂)
