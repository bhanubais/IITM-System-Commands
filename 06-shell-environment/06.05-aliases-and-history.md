# 06.05 Aliases and History

## Purpose

Create command shortcuts and recall previous commands.

Aliases improve speed, but they can also hide risky behavior.

## Strict Syntax

```bash
alias
alias name='command'
unalias name
\command
history
!!
!<number>
!$
```

## Command Map

| Command | Use |
|---|---|
| `alias` | List aliases. |
| `alias name='command'` | Create alias. |
| `unalias name` | Remove alias. |
| `\command` | Bypass alias for one invocation. |
| `source file` | Execute file in current shell. |
| `. file` | POSIX shorthand for `source`. |
| `history` | List command history. |
| `!!` | Run previous command. |
| `!N` | Run history entry number `N`. |
| `!$` | Reuse last argument of previous command. |
| `last` | Show login history. |

## Atomic Examples

```bash
alias
alias ll='ls -lGh'
alias rm='rm -i'
\ls
\rm -rf /tmp/cache/*
unalias ll
source ~/.bashrc
. ~/.bashrc
history
history 20
!!
!145
!$
last -1 "$USER"
```

## Compound Scenarios

Create defensive removal behavior:

```bash
alias rm='rm -i'
rm old.txt
```

Create a directory and enter it using history expansion:

```bash
mkdir project_core
cd !$
```

Audit current shell shortcuts:

```bash
alias
type ls
type rm
```

## Exam Traps

Aliases affect interactive command resolution.

Backslash bypasses an alias for that command execution.

`source` changes the current shell environment.

History expansion can execute unexpected commands if the wrong number is used.

## Production Notes

Do not depend on aliases inside scripts.

Use aliases to add safety prompts, not to hide destructive behavior.

Avoid putting passwords, tokens, or private keys in command history.
