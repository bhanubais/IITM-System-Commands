# 05.05 Awk Data Processing

The `awk` processor computes logic structures and executes block modifications far beyond standard text filtering constraints. It functions natively as an independent programming script.

## Real World Scenario: User Enumeration
During penetration testing or server auditing, you must quickly enumerate all user accounts installed on the system that have valid login shells. The `/etc/passwd` file stores this data separated by colons (`:`).
1. Query the file, setting the Field Separator (`-F`) to a colon.
2. Instruct the `awk` script natively to print the first column (Username) conditionally if the final column (Shell) matches `/bin/bash`:
   `awk -F":" '$7 == "/bin/bash" {print $1}' /etc/passwd`

## Structural Operations
AWK leverages record sequences. By default, integer records parse as single positional lines, segmented structurally by internal field definitions.

* `awk '{print $1}' <filename>` : Strip internal fields mapping exactly to the first sequence node (`$1`).
* `awk -F":" '{print $1}' /etc/passwd` : Modify the Internal Field Separator (`-F":"`). Output strictly the first positional field delimited by a colon.

## Advanced Data Architectures
Engineer comprehensive calculations directly on the output stream. Construct independent script files by prefixing the header with the `awk` interpreter (`#!/usr/bin/awk -f`) instead of forcing explicit inline execution.

* Execute `myscript.awk` directly by injecting target files via command line invocation: `./myscript.awk /etc/passwd`.

### Using AWK script files and libraries
Write reusable AWK programs as script files and invoke them with `-f`. You can stack multiple `-f` files: the order matters (libraries first, main script last).

Example: library file `mylib.awk` and main `myscript.awk`:
```awk
# mylib.awk
function greet(name) { printf("Hello %s\n", name) }
```

```awk
#!/usr/bin/awk -f
# myscript.awk
@include "mylib.awk"    # Not POSIX; use -f ordering instead for portability
{ greet($1) }
```

Run combining library and script (portable):
```bash
awk -f mylib.awk -f myscript.awk input.txt
# or make the main script executable with a shebang
chmod +x myscript.awk
./myscript.awk input.txt
```

Notes:
- `gawk` supports many extensions over POSIX `awk`; prefer `gawk` features only when targeting GNU systems.
- Avoid piping large files into `awk -f` unnecessarily; prefer passing filenames as arguments to allow `awk` to manage I/O efficiently.
