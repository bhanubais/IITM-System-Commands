# 07.01 Variables, Input and Output

## Purpose

Store script state.

Read user input.

Print predictable output.

Use this file for variable creation, arrays, `read`, `printf`, and special shell parameters. Use [07.06 Parameter Expansion and Quoting](07.06-parameter-expansion-and-quoting.md) for slicing, defaults, substitution, and quoting rules.

## Strict Syntax

```bash
name=value
declare -a array
declare -A map
read -r variable
printf 'format\n' "$value"
```

No spaces around `=`.

Quote variable expansions unless word splitting is required.

## Command Map

| Construct | Syntax | Operational Use |
|---|---|---|
| Scalar variable | `APP_ENV=prod` | Store one string. |
| Export | `export APP_ENV` | Pass variable to child processes. |
| Remove | `unset APP_ENV` | Delete active shell variable. |
| Read input | `read -r name` | Read one line without backslash parsing. |
| Prompt input | `read -r -p 'Name: ' name` | Prompt and capture interactively. |
| Print output | `printf '%s\n' "$name"` | Stable formatted output. |
| Integer variable | `declare -i count=0` | Force arithmetic assignment. |
| Indexed array | `arr=(alpha beta)` | Store ordered values. |
| Associative array | `declare -A ports=([ssh]=22)` | Store key-value pairs. |

## Atomic Examples

```bash
name="app01"
printf '%s\n' "$name"
export PATH="$PATH:/opt/tools/bin"
unset name
read -r line
read -r -p 'Target: ' target
printf 'user=%s uid=%s\n' "$USER" "$UID"
declare -i retries=3
declare -a hosts=(web01 web02 db01)
printf '%s\n' "${hosts[0]}"
printf '%s\n' "${hosts[@]}"
printf '%s\n' "${#hosts[@]}"
unset 'hosts[1]'
declare -A ports=([ssh]=22 [http]=80 [https]=443)
printf '%s\n' "${ports[ssh]}"
printf '%s\n' "${!ports[@]}"
```

## Special Parameters

| Parameter | Meaning |
|---|---|
| `$0` | Script or shell name. |
| `$1` | First positional argument. |
| `$#` | Number of positional arguments. |
| `$@` | All positional arguments, preserved when quoted. |
| `$?` | Exit status of previous command. |
| `$$` | PID of current shell. |

## Compound Scenarios

Capture a deployment target and refuse empty input:

```bash
read -r -p 'Deploy target: ' target
test -n "$target" || { printf '%s\n' 'target required' >&2; exit 1; }
printf 'deploying=%s\n' "$target"
```

Read hosts into an array and probe them:

```bash
hosts=(web01 web02 db01)
for host in "${hosts[@]}"; do
	printf 'checking %s\n' "$host"
	ping -c 1 "$host" >/dev/null 2>&1 || printf 'failed=%s\n' "$host"
done
```

Map service names to ports:

```bash
declare -A ports=([ssh]=22 [http]=80 [https]=443)
for service in "${!ports[@]}"; do
	printf '%s %s\n' "$service" "${ports[$service]}"
done | sort
```

## Exam Traps

`name = value` is not assignment. It is parsed as a command.

`echo` behavior varies across shells. Use `printf` for strict output.

`$@` and `$*` are not equivalent when quoted.

Arrays are Bash features. POSIX `sh` does not support indexed arrays.

## Production Notes

Use `read -r` by default.

Use `printf` for logs and generated config.

Avoid global variable names such as `PATH`, `USER`, and `HOME` unless intentionally modifying shell state.
