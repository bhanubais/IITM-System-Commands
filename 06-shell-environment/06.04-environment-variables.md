# 06.04 Environment Variables

System parameters load explicit structural coordinates dynamically into shell routines. Engineer global limits mapping logic across terminal instances.

## Core Variable Audit
* `printenv` : Extract the complete matrix of exported global memory variables.
* `env` : Dump the identical execution parameters inherited by child routines.
* `set` : Generate all defined user memory arrays, including explicitly un-exported localized functions and constants.
* `echo $<VARIABLE>` : Output an isolated value linked to the structural variable payload.

## Default Shell Parameter Architecture

| Variable Name | Execution Boundary |
| :--- | :--- |
| `DISPLAY` | Define graphic interface execution targets. |
| `HOME` | Explicit home coordinate path (e.g., `/home/kanika`). |
| `LOGNAME` | Structural authenticated username sequence. |
| `MAIL` | Incoming communication path arrays. |
| `PWD` | Map absolute present working directory path. |
| `OLDPWD` | Buffer memory targeting the directory logically prior to latest `cd` operation. |
| `PATH` | Absolute colon-separated directory arrays mapping executable targets for commands. |
| `SHELL` | Static target pointing exactly to the shell node binary (`/bin/bash`). |
| `TERM` | Render physical syntax type logic (e.g., `xterm`). |
| `USER` | Strict login name execution pointer. |

## Identity Reconnaissance
* `id` : Dump precise identity architecture matching running configurations.
* `whoami` : Isolate current execution level parameter integer to base username strings.
* `who` : Analyze specific network users mapped into active shell arrays. `who -a -H` computes explicit header logic output parameters.
