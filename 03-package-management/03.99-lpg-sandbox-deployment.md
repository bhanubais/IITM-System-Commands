# 03.99 LPG Sandbox Deployment

## Purpose

Track optional Linux Pocket Guide sandbox setup.

This is lab infrastructure, not production infrastructure.

## Strict Syntax

```bash
curl -O <url>
tar -xf <archive>
bash reset-lpg
```

## Command Map

| Command | Use |
|---|---|
| `curl -O` | Download using remote filename. |
| `tar -xf` | Extract archive. |
| `bash reset-lpg` | Reset the practice sandbox. |
| `git checkout -- .` | Discard sandbox changes when it is a Git repository. |

## Atomic Examples

```bash
curl -O https://linuxpocketguide.com/LPG4.tar.gz
tar -xf LPG4.tar.gz
bash reset-lpg
git checkout -- .
```

## Compound Scenarios

Download and initialize the sandbox:

```bash
mkdir -p ~/labs/lpg
cd ~/labs/lpg
curl -O https://linuxpocketguide.com/LPG4.tar.gz
tar -xf LPG4.tar.gz
```

Reset a sandbox checkout:

```bash
cd ~/linuxpocketguide
git checkout -- .
bash reset-lpg
```

Verify archive extraction before reset:

```bash
tar -tf LPG4.tar.gz | head
tar -xf LPG4.tar.gz
```

## Exam Traps

This page is optional lab setup, not core package-management syntax.

`curl -O` keeps the remote filename.

`git checkout -- .` discards local changes in the current repository.

## Production Notes

Run this only in an isolated local folder.

Do not execute third-party reset scripts on production hosts.

Inspect downloaded archives before trusting their contents.
