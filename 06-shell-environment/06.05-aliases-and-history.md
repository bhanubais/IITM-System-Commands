# 06.05 Aliases and History

Regulate manual command environments mapping shortcuts and harvesting localized structural history.

## Real World Scenario: Defense Against Truncation
Accidentally running `rm *` in the wrong directory is a fatal engineering error. To protect execution bounds, administrators force defensive operations explicitly via aliases mapped in `.bashrc`. They establish `alias rm='rm -i'`, forcing the system kernel to request interactive `y/n` confirmation strictly for every file destruction. If the engineer truly needs to bypass the safeguard for a massive automated deletion, they prepend a backslash to temporarily revert to the raw binary payload: `\rm -rf /tmp/cache/*`.

## Output and Parameter Binding
* `alias` : Execute without parameters to output all globally mapped session aliases.
* `alias <name>="<command>"` : Substitute verbose parameter mappings securely behind single command tags. Example: `alias ll='ls -lGh'`.
* `\<command>` : Prepend a backslash to temporarily bypass active alias configurations. Example: `\ls` executes the fundamental `/bin/ls` binary independent of local `-color` augmentations.
* `unalias <name>` : Terminate the alias tag mapping explicitly.
* `source <file.sh>` : Read and parse specific commands within the file straight into active shell limits. It bypasses execution separation to update current environment variables. Alternate syntax utilizes simply `. <file.sh>`.

## Execution History
* `history` : Export the complete memory buffer of previously deployed shell instructions.
* `last` : Parse login activity logs. Output structure bounds to specific users via `last -1 $USER` or limit returns globally via `last -1`.
* `!!` : Execute the precise command sequence run immediately prior.
* `!145` : Trigger the command mapped to exact line integer 145 within the history log.
* `!$` : Extract and append the single final argument processed during the previous shell sequence. Useful for acting locally on files generated one second prior (e.g. `mkdir project_core` followed directly by `cd !$`).
