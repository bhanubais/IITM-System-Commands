# 03.03 Archival and Compression

Control static data aggregation and compress binary footprints locally prior to deployment layers.

## Standard Compression Algorithms
Linux relies upon localized stream compressors wrapping individual files.
* `gzip <file>` : Compress the specified node, stripping the original.
* `gunzip <file.gz>` : Decompress the GZIP payload.
* `xz <file>` : Deploy high-density compression algorithms executing heavier CPU calculations.
* `bzip2 <file>` : Alternate compression specification yielding dense block storage.

## Tape Archival Structure (tar)
Aggregate localized data trees into singular deployable payloads.
* `tar -cf archive.tar <dir>` : Compile the designated directory strictly into an uncompressed payload map.
* `tar -czf archive.tar.gz <dir>` : Compile and execute GZIP compression algorithms synchronously.
* `tar -xf archive.tar` : Extract the complete internal tree out into the working filesystem.
* `tar -xzf archive.tar.gz` : Extract and decompress immediately without intermediate file handling.

## Universal Archives
* `zip archive.zip <files>` : Deploy standard compression protocols ubiquitous across multi-OS networks.
* `unzip archive.zip` : Revert data out of generalized ZIP packages.
