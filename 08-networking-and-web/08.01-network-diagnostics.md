# 08.01 Network Diagnostics

Use these commands to see how a machine reaches other hosts and where connectivity breaks down.

## Real World Scenario: Port Exhaustion Troubleshooting
A web application abruptly stops serving database requests. The administrator executes `netstat -tulpn` directly in the shell. They observe the application's PID binding thousands of `TIME_WAIT` loops natively against external endpoints, identifying total TCP port exhaustion dynamically without checking isolated application GUI dashboards.

## Connection Parameters
* `ping <domain>` : Send ICMP probes to check whether a host responds. Stop with `Ctrl+C`.
* `traceroute <domain>` : Show each hop between your system and a target host.
* `netstat -tulpn` : List listening sockets and the processes that own them.

## Address Resolutions
* `ifconfig` or `ip addr show` : Show network interfaces and assigned IP addresses.
* `whois <domain>` : Look up registration details for a domain.
* `nslookup <domain>` : Query DNS records for a name.

Example: examine WHOIS output with a pager:
```bash
# View registrar and contact data safely
whois example.com | less
```
