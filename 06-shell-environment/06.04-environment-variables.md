# 06.04 Environment Variables

## Purpose

Inspect and modify values passed from the shell to child processes.

Environment variables control paths, identity, display settings, shell behavior, and tool configuration.

## Strict Syntax

```bash
printenv
env
set
echo "$VARIABLE"
export NAME=value
export PATH="$PATH:<directory>"
```

## Command Map

| Command or Variable | Use |
|---|---|
| `printenv` | Print exported environment variables. |
| `env` | Print environment passed to child commands. |
| `set` | Print shell variables, functions, and environment. |
| `export` | Mark variable for child processes. |
| `id` | Show user and group IDs. |
| `whoami` | Print current username. |
| `who` | Show logged-in users. |
| `$-` | Show active shell option flags. |

## Common Variables

| Variable | Meaning |
|---|---|
| `DISPLAY` | Graphical display target. |
| `HOME` | User home directory. |
| `LOGNAME` | Login name. |
| `MAIL` | Mailbox path. |
| `PWD` | Current directory. |
| `OLDPWD` | Previous directory. |
| `PATH` | Directories searched for commands. |
| `SHELL` | User shell path. |
| `TERM` | Terminal type. |
| `USER` | Current user name. |

## Atomic Examples

```bash
printenv
env
set
echo "$PATH"
echo "$HOME"
export APP_ENV=prod
export PATH="$PATH:$HOME/opt/python_tools/bin"
id
whoami
who
who -a -H
echo "$-"
```

## Compound Scenarios

Add a tool directory to `PATH`:

```bash
export PATH="$PATH:$HOME/opt/python_tools/bin"
command -v analyzer
```

Compare identity and environment:

```bash
whoami
id
printenv USER
```

Check whether the shell is interactive:

```bash
case "$-" in
	*i*) printf '%s\n' interactive ;;
	*) printf '%s\n' noninteractive ;;
esac
```

## Exam Traps

Shell variables are not automatically environment variables.

`export` makes a variable visible to child processes.

`PATH` is colon-separated.

`PWD` and `OLDPWD` are maintained by the shell.

## Production Notes

Modify `PATH` by appending or prepending directories deliberately.

Inspect environment when commands work interactively but fail in scripts or services.

Avoid storing secrets in broadly inherited environment variables unless the runtime requires it.
