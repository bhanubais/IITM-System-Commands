# 02.01 Directory Governance

Establish control over the filesystem hierarchy. Execute strict spatial navigation and directory state modifications.

## Spatial Navigation
* `pwd` : Output the present working directory. Alternatively use `echo $PWD`.
* `cd <path>` : Change the active directory context.
* `cd ..` : Traverse up to the parent directory.
* `cd ~` : Jump directly to the current user home directory.
* `cd /` : Jump directly to the root of the filesystem. The parent of root is root itself.

## Directory State Modifications
* `mkdir <dir_name>` : Construct a new directory node.
* `mkdir -p path/to/target/dir` : Engineer a complete directory tree. The `-p` flag forces the creation of all required parent directories.
* `rmdir <dir_name>` : Remove a directory strictly if it contains zero descendant nodes.

## Hierarchy Audits
* `ls` : List descendant structures in the current node.
* `ls -a` : Expose all items including hidden configuration files.
* `ls -l` : Output a strict long format detailing permissions, ownership, and sizes.
* `ls -ld <dir_name>` : Audit the directory node itself rather than its contents.
* `tree` : Map the hierarchy as a visual tree structure.
* `tree -d` : Filter the tree output to display only directory nodes.
