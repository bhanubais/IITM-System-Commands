# 06.04 Environment Variables

Environment variables carry settings from one shell session into another. They are how your shell remembers paths, names, and tool behavior.

## Real World Scenario: Path Augmentation
A developer installs a custom Python framework into `~/opt/python_tools/bin/`. When they type the tool name `analyzer`, the system outputs `command not found`. The kernel cannot execute the binary because the custom directory is not loaded into the strict `PATH` parameters. The developer natively appends the directory by exporting it securely into the shell history: `export PATH="$PATH:$HOME/opt/python_tools/bin"`. Now, `analyzer` resolves instantly from any working directory.

## Core Variable Audit
* `printenv` : Show exported environment variables.
* `env` : Print the environment passed to child processes.
* `set` : Show shell variables, functions, and environment values.
* `echo $<VARIABLE>` : Print a single variable value.

## Default Shell Parameter Architecture

| Variable Name | Execution Boundary |
| :--- | :--- |
| `DISPLAY` | Define graphic interface execution targets. |
| `HOME` | Explicit home coordinate path (e.g., `/home/kanika`). |
| `LOGNAME` | Structural authenticated username sequence. |
| `MAIL` | Incoming communication path arrays. |
| `PWD` | Map absolute present working directory path. |
| `OLDPWD` | Buffer memory targeting the directory logically prior to latest `cd` operation. |
| `PATH` | Absolute colon-separated directory arrays mapping executable targets for commands. |
| `SHELL` | Static target pointing exactly to the shell node binary (`/bin/bash`). |
| `TERM` | Render physical syntax type logic (e.g., `xterm`). |
| `USER` | Strict login name execution pointer. |

## Identity Reconnaissance
* `id` : Show your user and group IDs.
* `whoami` : Print the current username.
* `who` : Show who is logged in. `who -a -H` adds more detail.

## Operational Execution Flags (`$-`)
The `$-` parameter shows the shell's current option flags.
* `h` : Hash commands.
* `B` : Brace expansion structures explicitly activated.
* `i` : Interactive shell profile active.
* `m` : Job control protocol logic explicitly enabled.
* `H` : Process exclamation (`!`) style history substitution arrays activated explicitly.
* `s` : Input streams actively polled directly cleanly parsing through `stdin`.
* `c` : Commands read natively explicitly through binary string passing arguments.
