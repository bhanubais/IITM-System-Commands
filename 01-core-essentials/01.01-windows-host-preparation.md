# 01.01: Windows Host Preparation

Before accessing the Linux kernel, the underlying Windows host must be provisioned with standard package management and shell interfaces. This module documents the deployment of the Windows Package Manager (`winget`) and PowerShell (`pwsh`), establishing the required foundation for subsequent Linux initialization.

## Windows Package Manager (winget)

The `winget` utility manages software installations natively within the Windows environment. It is imperative to periodically upgrade the manager itself before deploying other packages.

### Base Commands

Search the remote repository for a specific package:
```bash
winget search <package_name>
```

Display detailed deployment information for a target package:
```bash
winget show <package_id>
```

### Installation and Lifecycle Management

Execute standard deployment, upgrade, or removal operations:
```bash
winget install <package_id>
winget upgrade <package_id>
winget uninstall <package_id>
```

List all locally installed software managed by `winget`:
```bash
winget list --source winget
```

### Upgrading the Package Manager

To ensure the registry and executable are current, upgrade `winget` directly:
```bash
winget --version
winget list winget
winget upgrade winget
```

## PowerShell Operations (pwsh)

PowerShell provides the primary interface for managing Windows host services prior to invoking the Windows Subsystem for Linux (WSL).

Install the core PowerShell interface via `winget`:
```bash
winget install Microsoft.PowerShell
```

Verify the installation and upgrade it during standard maintenance cycles:
```bash
pwsh --version
winget upgrade Microsoft.PowerShell
```

Once the host environment is strictly provisioned with `winget` and `pwsh`, proceed to WSL initialization.
