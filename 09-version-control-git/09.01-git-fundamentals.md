# 09.01 Git Fundamentals

Operate sequential memory graphs tracking iteration arrays globally. Prevent codebase destructions natively without archiving duplicate physical folder states.

## Node Initialization and State Tracking
* `git init` : Initialize an empty monitoring array in the current working directory boundary.
* `git clone <URL>` : Mirror remote hardware vectors onto a local machine environment reliably.
* `git status` : Audit the untracked and modified payload strings.

## Staging and Commits
* `git add <file>` : Push single modified instances into localized staging buffers. Call `git add .` to push everything sequentially.
* `git commit -m "Message"` : Execute permanent sequence block lock committing staging elements into historical indexing permanently.

## Branch Matrix and Remote Data
* `git branch <name>` : Spool an independent engineering environment node isolating production data files.
* `git checkout <name>` : Traverse execution vectors natively jumping immediately to that specified target node parameter.
* `git remote add origin <URL>` : Link physical active logic servers against local repositories.
* `git push -u origin <branch>` : Transmit compiled blocks successfully against external systems.
* `git pull` : Refresh current local files extracting most upstream external iterations automatically.
