# 05.02 POSIX Regular Expressions

## Purpose

Match text patterns with POSIX Basic Regular Expressions and Extended Regular Expressions.

Regex syntax is shared across tools, but each tool chooses its engine and flags.

## Strict Syntax

```text
BRE: grep 'pattern' file
ERE: grep -E 'pattern' file
ERE: egrep 'pattern' file
```

## Command Map

| Token | Meaning |
|---|---|
| `.` | Match any single character except newline. |
| `*` | Match zero or more of the previous atom. |
| `[]` | Match one character from a set. |
| `[^set]` | Match one character not in the set. |
| `^` | Start of line anchor. |
| `$` | End of line anchor. |
| `\` | Escape metacharacter meaning. |
| `+` | ERE one or more. |
| `?` | ERE zero or one. |
| `|` | ERE alternation. |
| `{n,m}` | ERE repetition range. |
| `()` | ERE grouping. |

## Character Classes

| Class | Match Scope |
|---|---|
| `[[:alnum:]]` | Letters and digits. |
| `[[:alpha:]]` | Letters. |
| `[[:digit:]]` | Decimal digits. |
| `[[:space:]]` | Whitespace. |
| `[[:blank:]]` | Space or tab. |
| `[[:cntrl:]]` | Control characters. |

## Atomic Examples

```bash
grep '^error' app.log                 # line starts with error
grep 'done$' app.log                  # line ends with done
grep '^$' file.txt                    # empty lines
grep '[[:digit:]]' file.txt           # any decimal digit
grep '[^a-z]' file.txt                # any character outside lowercase a-z
grep -E '(error|warn)' app.log        # ERE alternation
grep -E '[[:digit:]]{1,3}' file.txt   # ERE repeat range
grep -E '[0-9]{1,3}(\.[0-9]{1,3}){3}' input.txt # IPv4-like pattern
grep '\([0-9]\)\1' input.txt          # BRE backreference for repeated digit
```

## Compound Scenarios

Extract IPv4-like strings:

```bash
grep -E '[0-9]{1,3}(\.[0-9]{1,3}){3}' payload.txt
```

Remove empty and comment lines:

```bash
grep -v '^$' app.conf | grep -v '^#'
```

Find repeated adjacent digits with BRE backreference:

```bash
grep '\([0-9]\)\1' numbers.txt
```

## Exam Traps

BRE and ERE treat grouping and repetition differently.

In BRE, grouping uses `\(` and `\)`.

In ERE, grouping uses `(` and `)`.

`[error]` matches one character from the set, not the word `error`.

## Production Notes

Use single quotes around regex patterns to avoid shell expansion.

Use character classes when locale behavior matters.

Test regex on sample input before running destructive pipelines.
