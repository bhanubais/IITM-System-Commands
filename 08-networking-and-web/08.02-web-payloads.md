# 08.02 Web Payloads and HTTP Interaction

## Purpose

Fetch HTTP payloads from the terminal.

Use these commands for downloads, API checks, and remote configuration retrieval.

## Strict Syntax

```bash
curl <url>
curl -o <local_file> <url>
curl -O <url>
curl -s -H '<header>' <url>
wget <url>
```

## Command Map

| Command | Use |
|---|---|
| `curl URL` | Print response body to stdout. |
| `curl -o file URL` | Save response to chosen filename. |
| `curl -O URL` | Save using remote filename. |
| `curl -s` | Silent mode. |
| `curl -H` | Add request header. |
| `wget URL` | Download URL to local file. |

## Atomic Examples

```bash
curl https://example.com
curl -o target.tar.gz https://example.com/file.gz # choose local filename
curl -O https://example.com/file.gz   # keep remote filename
curl -s https://example.com/health    # silent progress, body still prints
curl -s -H "Authorization: Bearer $TOKEN" https://api.internal.org/config.json # add header
wget https://example.com/file.gz
```

## Compound Scenarios

Fetch remote configuration with a bearer token:

```bash
curl -s -H "Authorization: Bearer $TOKEN" https://api.internal.org/config.json > config.json
```

Download and verify an archive name:

```bash
curl -O https://example.com/release.tar.gz
ls -l release.tar.gz
```

Save a payload under an explicit filename:

```bash
curl -o agent.iso https://example.com/downloads/system-agent.iso
sha256sum agent.iso
```

## Exam Traps

`curl URL` prints to stdout by default.

`curl -o file URL` chooses the local filename.

`curl -O URL` uses the remote filename.

Silent mode hides progress output, not the response body.

## Production Notes

Redirect secrets and config payloads carefully.

Use checksum validation for downloaded binaries and archives.

Avoid printing sensitive API responses into shared terminal logs.
