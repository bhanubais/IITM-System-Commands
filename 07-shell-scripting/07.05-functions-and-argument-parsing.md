# 07.05 Functions and Argument Parsing

## Purpose

Group reusable shell logic and parse runtime flags.

Functions reduce duplication. `getopts` provides small command-line interfaces.

## Strict Syntax

```bash
name() {
	command
}

while getopts "ha:" opt; do
	case "$opt" in
		h) ... ;;
		a) value=$OPTARG ;;
		\?) ... ;;
	esac
done
```

## Command Map

| Construct | Use |
|---|---|
| `function_name()` | Define shell function. |
| `local name=value` | Create function-local variable in Bash. |
| `$1`, `$2` | Function positional parameters. |
| `return` | Return function status. |
| `getopts` | Parse short options. |
| `OPTARG` | Value passed to an option. |
| `OPTIND` | Index of next argument to parse. |
| `shift` | Remove positional arguments from the front. |

## Atomic Examples

```bash
hello() { printf 'hello\n'; }
hello
add() { local a=$1 b=$2; printf '%s\n' "$((a + b))"; } # local is Bash function scope
add 5 10
return 0                              # function status, valid inside function
while getopts "ha:" opt; do printf '%s\n' "$opt"; done # h flag, a flag requires argument
shift "$((OPTIND - 1))"               # remove parsed options from positional args
```

## Compound Scenarios

Define and call a calculation function:

```bash
calculate_payload() {
	local base=$1
	local limit=$2
	printf '%s\n' "$((base + limit))"
}
calculate_payload 5 10
```

Parse a required argument:

```bash
while getopts "ha:" opt; do
	case "$opt" in
		h) printf 'usage: script -a target\n'; exit 0 ;;
		a) target_arg=$OPTARG ;;        # argument attached to -a
		\?) printf 'invalid flag\n' >&2; exit 1 ;;
	esac
done
```

Wrap repeated API calls:

```bash
transmit_curl_payload() {
	local route=$1
	local json=$2
	curl -s -H "Authorization: Bearer $TOKEN" -d "$json" "https://api.example/$route"
}
```

## Exam Traps

Function parameters have their own `$1`, `$2`, and `$@`.

`local` is Bash, not POSIX `sh`.

`getopts` handles short options, not GNU long options by itself.

An option followed by `:` in the optstring requires an argument.

## Production Notes

Keep functions small and named by action.

Return status codes with `return`, print data through stdout.

Validate required arguments after `getopts`.
