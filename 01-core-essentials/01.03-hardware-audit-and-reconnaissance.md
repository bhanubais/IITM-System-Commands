# 01.03: Hardware Audit and Reconnaissance

Execute hardware audits to extract system metadata. Calibrate infrastructure visibility using structured commands.

## General System Identification

Extract global machine descriptors and operating system specifications.

* `uname -a` : Output the current machine architecture and OS details.
* `hostnamectl` : Audit the hostname, OS version, Kernel release, and Machine ID.
* `lsb_release -a` : Extract distribution-specific release information.

## Processor and Memory Validation

Audit core compute and memory allocations directly from the `/proc` filesystem.

* `cat /proc/cpuinfo` : Extract complete CPU specifications and processor counts. (Scale using `batcat` if installed).
* `cat /proc/meminfo` : Read raw memory configuration metrics.
* `free -h` : Display active, free, and swap RAM utilization in a human-readable format.

## Storage and Partition Governance

Monitor filesystem structures, storage boundaries, and active mount points.

* `df -h` : Audit hard disk partition capacity and usage percentage.
* `cat /proc/partitions` : Validate kernel-recognized disk partitions.
* `fdisk -l` : List precise partition table structures. Requires elevated privileges.

## Advanced Hardware Probing

Utilize specialized packages to probe and index granular hardware nodes. Install these tools via package management if missing.

* `lshw` : List hardware tree. Isolate specific components using classes. Example: `lshw -c display` or `lshw -c processor`.
* `hwinfo` : Execute a comprehensive hardware scan spanning PCI, graphics, keyboard, and block devices.
* `dmidecode` : Dump system hardware data directly from the DMI (SMBIOS) table.
* `lspci` : Audit all connected PCI buses and their corresponding internal devices. Provided by the `pciutils` package.

## Network Interfaces

* `ifconfig` : Audit ethernet, wireless, and loopback adapter configurations. Provided by the `net-tools` package.
