# 07.04 Command Redirection and Substitution

Engineer precise control over input and output execution channels. Route payload data strictly via POSIX standardized system components.

## Logical File Descriptors
* `0` : Assigned to Standard Input (`stdin`).
* `1` : Maps precisely to Standard Output (`stdout`).
* `2` : Regulates explicit error tracing Standard Error (`stderr`).

## Chained Command Sequencing
* `cmd1 ; cmd2` : Execute instructions consecutively independent of failure paths.
* `cmd1 && cmd2` : Execute `cmd2` safely only if `cmd1` operates successfully.
* `cmd1 || cmd2` : Trigger `cmd2` exclusively as an error mitigation reaction if `cmd1` suffers structural failure.

## Output Stream Relocation
* `>` : Overwrite the mapped target node totally capturing the active datastream.
* `>>` : Append payload cleanly ignoring existing file blocks and retaining data states.
* `2> fail.log` : Split operation stderr explicitly writing straight into localized debug tables.
* `&>` : Redirect the active stdout structure combined simultaneously with the stderr log structure.

## Linear Command Substitution
Inject nested execution strings evaluating algorithms recursively inside variables overriding text arrays.
* Deploy standard POSIX sequences `$(command)`. Reject archaic structural variants mimicking backticks logic loops.
* Example protocol: `echo "The runtime active server path matches $(pwd) structure."`.

## Complex Process Replacement Configurations
Specified binaries read internal execution from files strictly failing bare `stdin`. Reconstruct parameters using complex dynamic environments passing the process structure via `<( )`.
* Syntax implementation: `diff <(ls *.jpg) <(ls *.txt)`. This evaluates divergence mapping distinct execution states synchronously over parallel hardware threads.
