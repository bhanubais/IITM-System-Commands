# 04.04 ed Line Editor Automation

The `ed` utility acts as the original, standard UNIX line editor. Operational syntax adheres to the framework: `[address_1[,address_2]]command[parameters]`. While rarely used interactively today, it serves as the foundational logic behind `sed` and scriptable text manipulations.

## Real World Scenario: Scripted Text Processing
Because `ed` relies strictly on line commands rather than a visual buffer, it runs excellently inside isolated pipeline scripts where human interaction is impossible.

1. Launch `ed` safely on a target file: `ed server.conf`.
2. Delete lines 1 through 3 automatically: `1,3d`.
3. Save the modifications: `w`.
4. Quit: `q`.

## Fundamental Execution State
* Enter `e <filename>` to load a file into the active buffer.
* Execute `w <filename>` to commit the buffer data to disk.
* Enter `q` to quit the editor.
* Trigger `P` to toggle the prompt status indicator.
* Execute `!command` to bypass the editor and run an external shell calculation.
* Read external command outputs directly into buffer with `r !command`.

## Editing Operations
* `p` : Print active target line to standard output.
* `n` : Display the target line prefixed with its integer line number.
* `i` : Insert a new data block exactly before the targeted line.
* `a` : Append a new data block exactly after the targeted line.
* `c` : Change the contents of the entire targeted line block.
* `d` : Delete the specific targeted line block.
* `m <pos>` : Move targeted line array to a new structural index.
* `u` : Revert the strict latest mutation.

All data insertions (`i`, `a`, `c`) must be aggressively terminated by placing a lone `.` on an empty line. This signals to the parsing engine that the input stream is over.

## Regex Substitution
The core search-replace structure parses as `s/query/replacement/g`.
Prefixing a line boundary confines the substitution range safely.
* `3,5s/PREFIX/prefix/` : Execute discrete substitution for the string "PREFIX" exclusively across structural lines 3 through 5.

## Quick Reference
Compact ed quick commands for scripting contexts:

- Start: `ed filename`
- Print lines: `1` (first), `$` (last), `,p` (all)
- Delete: `d` (current), `1,3d` (range)
- Insert/append: `i` / `a` (terminate with lone `.`)
- Substitute: `s/old/new/` or `,s/old/new/g`
- Write and quit: `w` ; `q`

Notes:
- `ed` is script-friendly; use `ed -s` for silent, non-interactive scripts.
