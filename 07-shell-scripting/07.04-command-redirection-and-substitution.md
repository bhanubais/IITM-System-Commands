# 07.04 Command Redirection and Substitution

## Purpose

Route input, output, and errors deliberately.

Capture command output when a script needs a value, not a stream.

Use [07.07 Heredocs, Here-Strings, and Process Substitution](07.07-heredocs-here-strings-and-process-substitution.md) for heredocs, here-strings, FIFOs, and process substitution.

## Strict Syntax

```bash
command >file
command >>file
command 2>error.log
command >output.log 2>error.log
command &>combined.log
value="$(command)"
cmd1 && cmd2
cmd1 || cmd2
cmd1 ; cmd2
```

## Command Map

| Operator | Meaning |
|---|---|
| `>` | Redirect stdout, overwrite target. |
| `>>` | Redirect stdout, append to target. |
| `2>` | Redirect stderr. |
| `&>` | Redirect stdout and stderr in Bash. |
| `<` | Read stdin from file. |
| `$()` | Command substitution. |
| `;` | Run next command regardless of status. |
| `&&` | Run next command only after success. |
| `||` | Run next command only after failure. |

## File Descriptors

| Descriptor | Stream |
|---|---|
| `0` | stdin |
| `1` | stdout |
| `2` | stderr |

## Atomic Examples

```bash
date > run.log
date >> run.log
ls /missing 2> errors.log
rsync -a src/ dst/ > sync.out 2> sync.err
rsync -a src/ dst/ &> sync.log
sort < names.txt
today="$(date +%F)"
count="$(find . -type f | wc -l)"
mkdir -p logs && printf '%s\n' ready
test -f config.yml || printf '%s\n' 'missing config' >&2
```

## Compound Scenarios

Capture clean backup logs and failures separately:

```bash
stamp="$(date +%F-%H%M%S)"
mkdir -p logs
tar -czf "backup-$stamp.tar.gz" app/ >"logs/backup-$stamp.out" 2>"logs/backup-$stamp.err"
test -s "logs/backup-$stamp.err" && printf 'backup warnings recorded\n' >&2
```

Abort deployment when validation fails:

```bash
test -f app.conf && grep -q '^PORT=' app.conf && ./deploy.sh >deploy.log 2>deploy.err
status=$?
test "$status" -eq 0 || { printf 'deploy failed status=%s\n' "$status" >&2; exit "$status"; }
```

Capture command output for a report:

```bash
host="$(hostname)"
kernel="$(uname -r)"
files="$(find . -type f | wc -l)"
printf 'host=%s kernel=%s files=%s\n' "$host" "$kernel" "$files"
```

## Exam Traps

`>` overwrites.

`>>` appends.

`2>` captures stderr only.

Command substitution strips trailing newlines.

`for item in $(command)` breaks on whitespace. Use a read loop when lines matter.

## Production Notes

Always decide where stderr goes.

Quote command substitution assignments.

Use separate files for stdout and stderr when diagnosing failures.
