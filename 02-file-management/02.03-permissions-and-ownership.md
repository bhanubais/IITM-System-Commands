# 02.03 Permissions and Ownership

## Purpose

Control read, write, execute, ownership, and group access on filesystem nodes.

This page covers `chmod`, `chown`, and `chgrp`.

## Strict Syntax

```bash
chmod MODE target
chmod WHO+PERMS target
chmod WHO=PERMS target
chown USER target
chown USER:GROUP target
chgrp GROUP target
```

## Command Map

| Command | Use |
|---|---|
| `ls -l` | Audit permission string. |
| `chmod` | Change permission bits. |
| `chmod -R` | Apply permission changes recursively. |
| `chown` | Change owner. |
| `chown -R` | Change owner recursively. |
| `chgrp` | Change group. |

## Permission Model

| Layer | Meaning |
|---|---|
| User | File owner. |
| Group | Assigned group. |
| Others | Every other account. |

| Bit | File Meaning | Directory Meaning |
|---|---|---|
| `r` | Read file data. | List names. |
| `w` | Modify file data. | Create, rename, or delete entries. |
| `x` | Execute as program. | Traverse with `cd`. |

## Atomic Examples

```bash
ls -l app.sh
chmod u+x app.sh                      # add execute for owner
chmod g-w config.yml                  # remove write from group
chmod o=g file.txt                    # copy group permissions to others
chmod g+w,o+w shared.txt              # grant write to group and others
chmod a+rX docs/                      # read for all, traverse dirs only with X
chmod -R a+rX public/                 # apply recursively through tree
chmod 755 public_html/                # rwx r-x r-x
chmod 700 private/                    # owner only
chmod 644 index.html                  # rw- r-- r--
chmod 600 server.key                  # owner read/write only
chown nginx core.log
chown nginx:wheel core.log            # set owner and group together
chown -R dev_user:dev_group /var/www/html # recurse through web tree
chgrp developers shared_config.conf
```

## Compound Scenarios

Make a script executable and run it:

```bash
ls -l s1.sh
chmod 755 s1.sh
./s1.sh
```

Repair web tree ownership:

```bash
sudo chown -R www-data:www-data /var/www/html
sudo chmod -R a+rX /var/www/html
```

Lock down a private key:

```bash
chmod 600 server.key
ls -l server.key
```

## Exam Traps

The first character in `ls -l` is file type, not a permission bit.

Octal values are additive: read is `4`, write is `2`, execute is `1`.

Directory execute permission controls traversal.

Capital `X` grants execute only to directories or already executable files.

## Production Notes

Be careful with recursive permission changes.

Use least privilege for keys and secrets.

Changing ownership usually requires elevated privileges.
