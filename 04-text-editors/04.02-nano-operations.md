# 04.02 nano Operations

## Purpose

Perform quick terminal edits with visible shortcuts.

`nano` is useful when speed matters more than modal editing power.

## Strict Syntax

```bash
nano <file>
sudo nano <file>
```

Common key notation:

```text
Ctrl+O
Enter
Ctrl+X
```

## Command Map

| Shortcut | Use |
|---|---|
| `Ctrl+O` | Write buffer to disk. |
| `Ctrl+S` | Save in place. |
| `Ctrl+R` | Read another file into buffer. |
| `Ctrl+X` | Exit. |
| `Ctrl+K` | Cut line. |
| `Alt+6` | Copy line. |
| `Ctrl+U` | Paste. |
| `Alt+U`, `Alt+E` | Undo and redo. |
| `Ctrl+W` | Search forward. |
| `Ctrl+Q` | Search backward. |
| `Alt+R` | Search and replace. |
| `Ctrl+C` | Show cursor position. |

## Compound Scenarios

Create an environment file:

```bash
nano .env
```

Inside `nano`:

```text
Ctrl+O
Enter
Ctrl+X
```

Open a privileged config:

```bash
sudo nano /etc/hosts
```

Search and replace inside a buffer:

```text
Ctrl+W
Alt+R
Ctrl+O
```

## Exam Traps

`Ctrl+O` writes the file. It does not exit.

`Ctrl+X` exits and may prompt when unsaved changes exist.

`Ctrl+C` reports cursor position inside `nano`.

## Production Notes

Use `nano` for small edits and direct config updates.

Avoid typing secrets into shell commands that enter history.

Use cursor position output when matching error line numbers.
