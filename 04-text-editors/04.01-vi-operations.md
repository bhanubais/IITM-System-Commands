# 04.01 vi Operations and Configuration Management

The `vi` editor operates through a strict tri-state structure. Absolute awareness of the current operational mode is required to successfully edit files over a network connection securely.

## Real World Scenario: Emergency Server Patching
When a critical vulnerability is detected on a remote server over SSH, graphical editors are unavailable. You must patch the `/etc/ssh/sshd_config` file natively.

1. Open the file: `sudo vi /etc/ssh/sshd_config`
2. Search for the port configuration: Type `/Port 22` and press Enter.
3. Switch to Insert Mode to change the port: Press `i`.
4. Escape back to Command Mode: Press `Esc`.
5. Save the configuration and force write: Type `:wq!` and press Enter.

## Tri-State Architecture
1. **Command Mode (`Esc`)**: The default initialization state. Keys execute structural commands (navigation, deletion, copy).
2. **Insert Mode (`i`, `o`, `a`)**: Character input state where keystrokes parse directly into the buffer.
3. **Ex Mode (`:`)**: Executes lowest-level file and environmental directives.

## Mode Transitions to Insert
* `i` : Insert at cursor position.
* `I` : Insert at the strict beginning of the line.
* `a` : Append explicitly after the cursor.
* `A` : Append at the extreme end of the current line.
* `o` : Open a new line directly below the cursor.
* `O` : Open a new line directly above the cursor.

## Navigation Governance
* Axis Control: `h` (Left), `j` (Down), `k` (Up), `l` (Right).
* `0` : Snap to line beginning.
* `$` : Snap to line termination.
* `w` / `b` : Jump forward or backward by one word block.
* `1G` or `:0`: Jump to absolute file beginning.
* `G` or `:$`: Jump to absolute file termination.
* `nG` or `:n`: Jump to absolute line `n`.

## Text Manipulation (Command Mode)
* `r` : Overwrite a single character natively without entering insert mode.
* `x` : Delete a single character.
* `dw` : Sever a single word.
* `dd` : Delete the entire line. Prefix with an integer integer to scale (e.g., `5dd` deletes 5 lines).
* `yy` : Yank (copy) the current line. Scale with numeric prefixes (e.g., `5yy`).
* `p` : Paste yanked or deleted blocks below the cursor.
* `u` : Undo previous operation.

## Environment Ex Controls
* `:w`: Write buffer to disk.
* `:q!`: Force quit, purging all unsaved modifications safely if a mistake was made.
* `:wq` or `:x`: Write modifications and exit securely.
* `:se nu` / `:se nonu`: Toggle visual line numbering. Extremely useful when debugging line-bound compiler errors.
* `:<range>s/old/new/g`: Execute substitutions. Example: `1,5s/line/LINE/g` to replace globally across lines 1 through 5.
