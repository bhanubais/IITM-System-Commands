# 05.07 Sort, Uniq, and Ranking

## Purpose

Order records.

Group repeated lines.

Rank counts for operational triage.

## Strict Syntax

```bash
sort file
sort -n file
sort -r file
sort -k FIELD,FIELD -t DELIM file
uniq file
uniq -c file
uniq -d file
```

## Command Map

| Command | Purpose |
|---|---|
| `sort` | Lexicographic ordering. |
| `sort -n` | Numeric ordering. |
| `sort -r` | Reverse ordering. |
| `sort -nr` | Numeric descending ordering. |
| `sort -k 2,2` | Sort by field two only. |
| `sort -t ':' -k 3,3n` | Sort colon-delimited input by numeric field three. |
| `uniq` | Collapse adjacent duplicate lines. |
| `uniq -c` | Count adjacent duplicate lines. |
| `uniq -d` | Print duplicated lines only. |

## Atomic Examples

```bash
sort names.txt
sort -r names.txt                    # reverse order
sort -n sizes.txt                    # numeric order
sort -nr scores.txt                  # numeric reverse order
sort -k 5,5 -t ' ' files.txt         # sort only by field 5
sort -t ':' -k 3,3n /etc/passwd      # colon delimiter, numeric UID field
sort names.txt | uniq
sort names.txt | uniq -c             # count adjacent duplicates after sort
sort names.txt | uniq -d             # print duplicated lines only
sort access.log | uniq -c | sort -nr # rank repeated log lines
```

## Compound Scenarios

Rank noisy IP addresses:

```bash
cut -d ' ' -f1 access.log | sort | uniq -c | sort -nr | head -n 10
```

Find duplicated package names in a list:

```bash
sort packages.txt | uniq -d
```

Rank files from an `ls -l` capture by size:

```bash
ls -l | tr -s ' ' | sort -k5,5nr -t ' ' | head -n 10
```

## Exam Traps

`uniq` only compares adjacent lines. Sort first unless input is already grouped.

`sort -n` treats fields as numbers.

`sort -k 5` and `sort -k 5,5` are not the same. The second restricts sorting to one field.

## Production Notes

Ranking pipelines usually follow this pattern:

```bash
extract | sort | uniq -c | sort -nr | head
```

Use field-specific `sort -k` when handling tabular logs or reports.
