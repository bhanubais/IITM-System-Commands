# 04.03 emacs Workspace Architecture

The `emacs` editor framework operates extensively through complex key chording. It is less an editor and more a complete operating environment heavily used by programmers.

## Real World Scenario: Multitasking Code Deployment
When compiling a dense C program, an engineer requires constant context switching without closing the editor. Emacs provides a suspended state workflow.

1. Open the source file: `emacs main.c`
2. Perform edits using key chords.
3. Rapidly suspend the editor without terminating it: `Ctrl`+`z`. This drops the user back to the shell prompt.
4. Compile the code: `gcc main.c`
5. Restore the editor seamlessly from the background: `fg`

## System State
* `Ctrl`+`x` followed by `Ctrl`+`s` : Save buffer stream to file.
* `Ctrl`+`x` followed by `Ctrl`+`c` : Terminate the `emacs` process entirely.
* `Ctrl`+`z` : Suspend `emacs` and return to shell without process termination.

## Spatial Positioning
* `Ctrl`+`a` / `Ctrl`+`e` : Anchor cursor at line beginning or termination.
* `Alt`+`<` / `Alt`+`>` : Anchor cursor at absolute file beginning or termination.
* `Ctrl`+`f` / `Ctrl`+`b` : Navigate forward or backward exactly one character.
* `Alt`+`f` / `Alt`+`b` : Navigate forward or backward exactly one word array.
* `Ctrl`+`n` / `Ctrl`+`p` : Navigate downward or upward exactly one line.
* `Ctrl`+`v` / `Alt`+`v` : Page downward or upward exactly one screen viewport.

## Search and Buffer Operations
* `Ctrl`+`k` : Cut data from cursor to the exact line termination.
* `Alt`+`d` : Cut the word array immediately following the cursor.
* `Ctrl`+`y` : Paste (yank) data from internal memory at the cursor position.
* `Ctrl`+`s` : Initialize forward string query.
* `Ctrl`+`r` : Initialize backward string query.
* `Alt`+`x` : Shift into command input for operations like string replacement.
