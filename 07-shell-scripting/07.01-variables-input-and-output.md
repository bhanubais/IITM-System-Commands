# 07.01 Variables, Input and Output

Engineer dynamic memory configurations across scripts using structured shell variables. Manage internal storage efficiently without arbitrary global pollution.

## Real World Scenario: Automated Variable Provisioning
A deployment script requires an absolute path to a configuration directory. If the user explicitly provides `--config /opt/app/`, the script accepts the target loop. However, running the script entirely without arguments must not crash the build pipeline. The engineer uses Native Shell Fallback expansion (`${var:-default}`) to set a safe implicit state: `TARGET_DIR="${1:-/etc/default/app}"`. If `$1` is completely empty, it defaults automatically perfectly to `/etc/default/app`.

## Standard Input and Output
Capture user parameters iteratively via standard input and pipe values reliably to standard output. Vital for continuous algorithmic validation environments.

* `read -p "Prompt: " VAR` : Output prompt string and map subsequent keyboard entry stream directly to the defined `VAR`.
* `read -a arr` : Extract standard input array sequentially mapped by space delimiters into the shell parameter payload.
* `echo $VAR` : Stream assigned parameter string output explicitly to terminal display buffers.
* `echo -e "Line\nBreak"` : Parse escape sequence parameters strictly measuring internal structural breaks.
* `printf "Output: %s\n" "$VAR"` : Render strict POSIX C-style type string formats bypassing explicit echo environment assumptions.

## Core Definitions

* `MYVAR="string"` : Initialize a new variable. Spaces around the assignment operator are forbidden.
* `export MYVAR` : Export the configuration variable to spawn child shell environments globally.
* `unset MYVAR` : Purge the variable block from active memory.

## Restricting value types

- `declare -i myvar`: Only integer assigned
- `declare -l myvar`: Only lowercase characters assigned
- `declare -u myvar`: Only UPPERCASE characters assigned
- `declare -r myvar`: variable will be readonly

## Removing Restrictions

- `declare +i myvar`: integer restriction removed
- `declare +l myvar`: lowercase characters restriction removed
- `declare +u myvar`: UPPERCASE characters restriction removed
- `declare +r myvar`: Can't do once it is readonly.

## Indexed Arrays

- `declare -a arr` : declare an array
- `$arr[0]="value_at_1"` : set value at 0 position
- `arr+=("value")` : Append a new value

- `echo ${arr[0]}` : Display value at index 0
- `echo ${#arr[@]}` : Number of elements in the array i.e. length of array
- `echo ${!arr[@]}` : Display all indices used i.e. all elements
- `echo ${arr[@]}` : Display value of all elements in arr

- `unset 'arr[2]'` : Delete element at 2nd index

## Associative Arrays (Dictionaries)

- `declare -A hash` : Declare a hash dictionary
- `$hash["a"]="value"`: set value for key "a"

- `echo ${hash["a"]}` : Display value at key "a"
- `echo ${#hash[@]}` : Number of elements in the hash i.e. length of hash
- `echo ${!hash[@]}` : Display all indices used i.e. all elements
- `echo ${hash[@]}` : Display value of all elements in hash

- `unset 'arr["a"]'` : Delete element with key as "a"


## Special Built-in Parameters
* `$0` : Read the execution base name of the shell process.
* `$$` : Audit the Process ID integer of the active shell.
* `$?` : Extract the fundamental exit status code of the preceding system command.

## Test if a variable is set

1. `[[ -v myvar ]]`
2. `$?`

Output:
 - 0: success (variable is set)
 - 1: failure (variable is not set)

## Get or Set a default value of a variable

If the variable `myvar` is not set, **Display** "default" as its default value.
- `echo ${myvar:-"default"}`

If the variable `myvar` is not set, **Set and Display** "default" as its default value.
- `echo ${myvar:="default"}`

If the variable `myvar` is set, then **Set** "default" as its value.
- `echo ${myvar:+"default"}`

## List all matching variable names

- `echo ${!H*}`: List all variables starting with the letter "H"

## String Slicing and Pattern Transformation
* `${#var}` : Output the strict mathematical length integer of the block.
* `${var:5:4}` : Calibrate slice range. Bypass the first 5 characters and extract exactly 4 characters sequentially.
* `${var,,pattern}` : Global shift matching structural patterns to lowercase vectors exclusively entirely.
* `${var,pattern}` : Execute localized single shift mapping lowercase targets exactly exclusively one instance completely securely strictly natively.
* `${var^^pattern}` : Target mapping matches global upper-case transformation blocks definitively absolutely exactly.
* `${var^pattern}` : Transform specific target matching structurally uniquely entirely distinctly exclusively single mapping upper-case explicitly locally accurately.
* `${var^^[aeiou]}` : Force all matching vowel characters contained strictly within bounding group limits arrays instantly identically cleanly uppercase formats identically exactly securely.

## Substring Extraction
* `${var#pattern}` : Strip the shortest matching pattern array anchoring from the front boundary.
* `${var##pattern}` : Strip the absolute longest matching pattern from the front.
* `${var%pattern}` : Strip the shortest pattern block anchoring from the rear boundary.
* `${var%%pattern}` : Strip the specific maximum pattern matching from the rear list.

## Substring Substitution
* `${var/pattern/new}` : Substitute solely the first verified occurrence block.
* `${var//pattern/new}` : Substitute all verified match occurrences globally.
* `${var/#pattern/new}` : Execute substitution exclusively if the initial pattern aligns at the start boundary.
* `${var/#pattern/new}` : Match at beginning and replace with new.
* `${var/%pattern/new}` : Match at the end and replace with new.

## Fallbacks and Conditionals
* `${var:-default}` : Inject a default payload securely if the initial variable lacks instantiation.
* `${var:=default}` : Assign and inject the default payload string if the variable points null.
* `${var:+default}` : Trigger specific alternative output only if the base variable is explicitly populated.

## Context Case Modifications
* `${var,}` / `${var,,}` : Mutate the exact first character or the entire string data into lowercase architecture.
* `${var^}` / `${var^^}` : Mutate the precise first character or entire defined string into uppercase formatting.

## Array Initialization
* `arr=(2 4 6)` : Define an explicit numerical logic array.
* `echo ${arr[@]}` : Unpack and stream every internal array node safely.

## Using `mapfile` / `readarray` to populate arrays
`mapfile` (Bash) and its synonym `readarray` load lines from a file or command directly into an indexed array. They are fast and avoid common pitfalls with word-splitting.

Examples:
```bash
# Read lines from a file into an array (strip trailing newlines with -t)
mapfile -t lines < file.txt

# Read command output into an array
mapfile -t results < <(some-command --multiline)

# Equivalent using readarray (same behavior)
readarray -t items < <(printf '%s\n' "one" "two" "three")

# Iterate safely over elements
for i in "${!lines[@]}"; do
	printf '%s\n' "${lines[i]}"
done

# Join array elements into a single string (space-separated)
joined="${lines[*]}"
printf '%s\n' "$joined"
```

Notes and portability:
- `mapfile` / `readarray` are Bash builtins and not POSIX. For portable scripts use a safe read loop:
```bash
arr=()
while IFS= read -r line; do
	arr+=("$line")
done < file.txt
```
- `-t` strips trailing newlines from each line. Without `-t` lines include the terminating newline.
- `mapfile` loads the entire input into memory. Avoid it with very large streams; prefer streaming loops when memory is constrained.
