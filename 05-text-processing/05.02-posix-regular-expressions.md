# 05.02 POSIX Regular Expressions

Regular expressions let you match patterns in text. POSIX defines two common forms: Basic Regular Expressions (BRE) and Extended Regular Expressions (ERE).

## Real World Scenario: Extracting Threat Vectors
During a hackathon, you capture a massive raw data payload containing garbage strings intermixed with valid IPv4 addresses belonging to the threat actor. Writing a script to extract `###.###.###.###` is impossible using absolute strings. You must construct a regex sequence natively: `[0-9]{1,3}\.[0-9]{1,3}\.[0-9]{1,3}\.[0-9]{1,3}`. The ERE engine matches exactly that matrix dynamically.

## Metacharacter Governance

| Character | Rule |
| :--- | :--- |
| `.` | Match any single character except null or newline. |
| `*` | Match zero or more consecutive executions of the preceding block. |
| `[]` | Limit match exclusively to characters defined within the brackets. |
| `^` | Hard anchor binding the match to the strict beginning of the line. Also negates `[^set]` within brackets. Example: `^error` targets lines initiating with "error". |
| `$` | Hard anchor binding the match to the strict termination of the line. Example: `^$` targets completely empty lines instantly. |
| `\` | Escape metacharacter interpretation. |

## Bracket and Character Classes

| Class | Match Scope |
| :--- | :--- |
| `[[:alnum:]]` | Strictly alphanumeric `[A-Za-z0-9]`. |
| `[[:alpha:]]` | Strictly alphabetic `[A-Za-z]`. |
| `[[:digit:]]` | Strictly decimal digits `[0-9]`. |
| `[[:space:]]` | Target any form of whitespace (space, tab, newline). |
| `[[:blank:]]` | Target space or tab exclusively. |
| `[[:cntrl:]]` | Target invisible control characters. |

## Extended Engine Additions (ERE)

The extended engine calibrates multipliers and logic branching.

* `+` : The preceding character must appear one or more consecutive times.
* `?` : The preceding character must appear zero or exactly one time.
* `|` : Logical OR gate specifying alternative pattern block sets. Example: `(Anu|Raman)`.
* `{n,m}` : Range quantifier. Match the preceding pattern exactly between `n` and `m` times. Example: `[[:digit:]]{12}`.
* `()` : Define absolute operational groupings.

## Backreferences
Use backreferences to match the same text again later in the pattern. `\1` refers to the first captured group.
* Example: `\([0-9]\)\1` matches adjacent uniform digits, enforcing twin adjacent numbers (matching `33` inside IP variables but rejecting `34`).
* Example: `\(ma\).*\1` targets identical repetitions of `ma` bounding a dynamic string payload.
