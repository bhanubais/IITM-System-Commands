# 04.03 emacs Operations

## Purpose

Edit files in a programmable terminal editor with strong buffer and job-control behavior.

This page focuses on core movement, save, suspend, search, and edit keys.

## Strict Syntax

```bash
emacs <file>
fg
```

Inside `emacs`:

```text
Ctrl+x Ctrl+s
Ctrl+x Ctrl+c
Ctrl+z
```

## Command Map

| Shortcut | Use |
|---|---|
| `Ctrl+x Ctrl+s` | Save buffer. |
| `Ctrl+x Ctrl+c` | Exit Emacs. |
| `Ctrl+z` | Suspend Emacs. |
| `Ctrl+a`, `Ctrl+e` | Line start and end. |
| `Alt+<`, `Alt+>` | File start and end. |
| `Ctrl+f`, `Ctrl+b` | Forward and backward character. |
| `Alt+f`, `Alt+b` | Forward and backward word. |
| `Ctrl+n`, `Ctrl+p` | Next and previous line. |
| `Ctrl+v`, `Alt+v` | Page down and page up. |
| `Ctrl+k` | Cut to line end. |
| `Alt+d` | Cut next word. |
| `Ctrl+y` | Yank pasted text. |
| `Ctrl+s`, `Ctrl+r` | Forward and backward search. |
| `Alt+x` | Run named command. |

## Compound Scenarios

Suspend, compile, and return:

```bash
emacs main.c
```

Inside `emacs`:

```text
Ctrl+z
```

Back in shell:

```bash
gcc main.c
fg
```

Save and exit cleanly:

```text
Ctrl+x Ctrl+s
Ctrl+x Ctrl+c
```

Run a named command:

```text
Alt+x
replace-string
```

## Exam Traps

`Ctrl+z` suspends. It does not exit.

Use `fg` to return to the suspended editor.

Emacs uses key chords, often in sequence.

## Production Notes

Use suspend workflow when editing and compiling repeatedly.

Save before running external commands on the edited file.

Use search before large manual edits.
