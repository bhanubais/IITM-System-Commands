# 04.04 ed Line Editor

## Purpose

Edit files through line commands.

`ed` is useful in minimal systems and scripted editing contexts.

## Strict Syntax

```bash
ed <file>
ed -s <file>
```

Inside `ed`:

```text
[address[,address]]command[parameters]
w
q
```

## Command Map

| Command | Use |
|---|---|
| `e file` | Load file into buffer. |
| `w` | Write buffer. |
| `w file` | Write buffer to file. |
| `q` | Quit. |
| `P` | Toggle prompt. |
| `!command` | Run shell command. |
| `r !command` | Read command output into buffer. |
| `p` | Print line. |
| `n` | Print line with number. |
| `i`, `a`, `c` | Insert, append, or change text. |
| `d` | Delete addressed line or range. |
| `m <pos>` | Move addressed lines. |
| `u` | Undo last mutation. |
| `s/old/new/g` | Substitute text. |

## Atomic Examples

```text
1p
$p
,p
1,3d
i
new text
.
a
new text
.
s/old/new/
,s/old/new/g
w
q
```

## Compound Scenarios

Delete lines and save:

```bash
ed server.conf
```

Inside `ed`:

```text
1,3d
w
q
```

Script a substitution:

```bash
printf '%s\n' ',s/old/new/g' 'w' 'q' | ed -s app.conf
```

Read command output into a file:

```text
r !date
w
```

## Exam Traps

Inserted text for `i`, `a`, and `c` ends with a lone `.` on its own line.

`,` addresses the whole buffer.

`$` means the last line.

`ed -s` is silent and better for scripts.

## Production Notes

Use `ed` when only POSIX tools are available.

Prefer scripted `ed -s` for repeatable minimal edits.

Test command streams on a copy before editing critical files.
