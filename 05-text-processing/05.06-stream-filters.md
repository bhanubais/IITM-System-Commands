# 05.06 Stream Filters

## Purpose

Run small filters that reshape streams without full programming logic.

This file covers `cut`, `wc`, `tr`, and `seq`. Use [05.07 Sort, Uniq, and Ranking](05.07-sort-uniq-and-ranking.md) for ordering and de-duplication. Use [05.08 Paste, Tee, Xargs, and Delimited Data](05.08-paste-tee-xargs-and-delimited-data.md) for joins, fan-out, and argument conversion.

## Strict Syntax

```bash
cut -c LIST file
cut -d DELIM -f LIST file
wc -l file
tr SET1 SET2
tr -s SET
seq LAST
seq FIRST LAST
seq FIRST STEP LAST
```

## Command Map

| Command | Purpose |
|---|---|
| `cut -c` | Extract character positions. |
| `cut -d -f` | Extract delimiter-separated fields. |
| `wc -l` | Count lines. |
| `wc -w` | Count words. |
| `wc -c` | Count bytes. |
| `wc -m` | Count characters. |
| `tr` | Translate or delete characters. |
| `tr -s` | Squeeze repeated characters. |
| `seq` | Generate integer sequences. |

## Atomic Examples

```bash
cut -c 1-4 names.txt                 # extract character range
cut -c 1,3,5 names.txt               # extract selected character positions
cut -d ':' -f 1 /etc/passwd          # colon delimiter, field 1
cut -d ',' -f 1,3 data.csv           # comma delimiter, fields 1 and 3
wc -l access.log
wc -w README.md
wc -c payload.bin                    # byte count
printf 'A B C\n' | tr 'A-Z' 'a-z'    # translate uppercase to lowercase
printf 'a   b\n' | tr -s ' '         # squeeze repeated spaces
printf 'abc123\n' | tr -d '0-9'      # delete digits
seq 5
seq 1 2 9                            # start, step, end
seq -w 1 10                          # pad numbers to equal width
seq -s ',' 1 5                       # comma-separated sequence
```

## Compound Scenarios

Count shell users:

```bash
cut -d ':' -f 7 /etc/passwd | sort | uniq -c | sort -nr
```

Normalize space-heavy command output before field extraction:

```bash
ls -l | tr -s ' ' | cut -d ' ' -f5,9
```

Generate predictable test filenames:

```bash
for n in $(seq -w 1 10); do
	printf 'case-%s.log\n' "$n"
done
```

## Exam Traps

`cut -d ' ' -f5` fails on repeated spaces unless the stream is normalized first.

`wc -c` counts bytes. `wc -m` counts characters.

`seq` is external. Brace expansion is shell syntax, not the same mechanism.

## Production Notes

Use `cut` only when delimiters are stable.

Use `tr -s` before field extraction from space-aligned output.

Prefer `awk` when field logic becomes conditional.
