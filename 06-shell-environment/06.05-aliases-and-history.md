# 06.05 Aliases and History

Regulate manual command environments mapping shortcuts and harvesting localized structural history.

## Output and Parameter Binding
* `alias <name>="<command>"` : Substitute verbose parameter mappings securely behind single command tags. Example: `alias ll="ls -alF"`.
* `unalias <name>` : Terminate the alias tag mapping explicitly.
* `source <file.sh>` : Read and parse specific commands within the file straight into active shell limits. It bypasses execution separation to update current environment variables. Alternate syntax utilizes simply `. <file.sh>`.

## Execution History
* `history` : Export the complete memory buffer of previously deployed shell instructions.
* `!!` : Execute the precise command sequence run immediately prior.
* `!145` : Trigger the command mapped to exact line integer 145 within the history log.
* `!$` : Extract and append the single final argument processed during the previous shell sequence. Useful for acting locally on files generated one second prior (e.g. `mkdir project_core` followed directly by `cd !$`).
