# 06.02 Job Control and Processes

## Purpose

Move jobs between foreground, background, suspended, and detached states.

Audit and signal processes safely.

## Strict Syntax

```bash
command &
Ctrl+Z
jobs
bg %<job>
fg %<job>
disown %<job>
disown -h %<job>
ps aux
kill <pid>
echo "$?"
```

## Command Map

| Command | Use |
|---|---|
| `command &` | Start in background. |
| `Ctrl+Z` | Suspend foreground job. |
| `jobs` | List shell jobs. |
| `bg` | Resume job in background. |
| `fg` | Bring job to foreground. |
| `disown` | Remove job from shell job table. |
| `disown -h` | Prevent shell SIGHUP on exit while keeping job listed. |
| `top` | Live process view. |
| `ps` | Static process snapshot. |
| `kill` | Send signal to process. |

## Atomic Examples

```bash
sleep 300 &
jobs
jobs -l
bg %1
fg %1
disown %1
disown -h %1
top -d 1
ps auxw
ps aux --sort=-%cpu | head -n 6
ps aux --sort=-%mem | head -n 6
ps -ef | grep nginx
ps -u username
ps auxf
ps -eo pid,%cpu,%mem,cmd
ps -p 1234
kill 1234
echo "$?"
```

## Exit Codes

| Code | Meaning |
|---|---|
| `0` | Success. |
| `1` | General failure. |
| `2` | Misuse of shell syntax or command syntax. |
| `126` | Found but not executable. |
| `127` | Command not found. |
| `130` | Interrupted with `Ctrl+C`. |
| `137` | Killed with signal 9. |

## Compound Scenarios

Detach a long backup from an SSH session:

```bash
long_backup_command > /var/log/backup.log 2>&1 &
jobs -l
disown -h %1
```

Find high CPU processes:

```bash
ps aux --sort=-%cpu | head -n 6
```

Recover a broken terminal:

```text
Ctrl+J
reset
Ctrl+J
```

## Exam Traps

`Ctrl+Z` suspends. It does not terminate.

`bg` resumes in background. `fg` resumes in foreground.

`disown -h` does not redirect file descriptors.

Exit codes are reduced modulo 256.

## Production Notes

Redirect output before detaching long-running jobs.

Confirm PID before `kill`.

Use `tmux` for durable interactive operations when possible.
