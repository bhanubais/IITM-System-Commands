# 05.03 Text Search with Grep

`grep` evaluates text streams and outputs exact lines verifying the defined POSIX regex parameter.

## Real World Scenario: Unauthorized Intrusion Auditing
A critical system administrator suspects an unauthorized root login breach. Instead of scrolling through thousands of lines in `/var/log/auth.log` using a paged buffer, they isolate the violation string directly.
1. Run the targeted grep: `grep 'FAILED root' /var/log/auth.log`
2. Run a case-insensitive, reversed match to strip out local developer IPs dynamically: `grep -i 'failed' /var/log/auth.log | grep -v '192.168.1.50'`

## Executing Basic Matches (BRE)
The `pattern` variable utilized below represents a POSIX Regular Expression evaluated by the active search engine.

* `grep '[error]' /var/log/syslog` : Scan the log file explicitly targeting the literal string error boundary.
* `cat script.sh | grep 'function'` : Audit an incoming pipeline stream isolating specific algorithm blocks.
* `grep -i '^e' <filename>` : Toggle case-insensitivity. Target lines initiating with either `e` or `E`.
* `grep -v '^$' <filename>` : Execute reverse matching. Exclude lines consisting merely of `^$`, strictly stripping empty lines out of code outputs.
* `grep -c 'pattern' <filename>` : Transmit strictly the integer count of successfully matched lines without printing the payload.
* `grep -l 'pattern' <filename>` : Transmit strictly the name of the file generating a matched line.

## Executing Extended Matches (ERE)
Upgrade the regex parser to the Extended engine to unlock quantifiers (`+`, `?`) and structural logic (`|`).

* `egrep 'pattern' <filename>` : Invoke the Extended Regex Engine. Equivalent mechanism natively to `grep -E`.
* `egrep '(ED|ME)'` : Extract streams matching exact string variants "ED" or "ME" exclusively.
* `egrep '\b[[:alnum:]]+\.[[:alnum:]]+\b'` : Bound string segments representing raw URL formats utilizing the isolated word boundary marker `\b`.
