# 01.01 WSL Initialization

Configure the Windows Subsystem for Linux (WSL) environment. Ensure hardware virtualization is active at the BIOS level before execution.

## System Package Governance

Manage Windows host packages directly from the command line using the `winget` utility. Initialize these tools before deploying the Linux subsystem.

* `winget search <package_name>` : Locate packages in the public repository.
* `winget list <package_name>` : Audit installed packages.
* `winget install <package_id>` : Install required dependencies.
* `winget upgrade <package_id>` : Calibrate the package to the latest version. Update the manager itself using `winget upgrade winget`.

## WSL Architecture Operations

Deploy and maintain Linux distributions within the Windows host parameters.

* `wsl -l -v` : Audit installed distributions and their running versions.
* `wsl -l -o` : List distributions available for immediate deployment.
* `winget upgrade Microsoft.WSL` : Force update the core WSL architecture via the Windows package manager.

## Deployment and Upgrades

* `wsl --install` : Deploy the default Ubuntu LTS distribution.
* `wsl --install -d <DistroName>` : Install a specific target distribution.
* `wsl --update` : Fetch and apply the latest WSL kernel updates.
