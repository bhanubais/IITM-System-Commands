# 01.02: WSL Initialization

Configure the Windows Subsystem for Linux (WSL) environment. Hardware virtualization must be active at the BIOS level before proceeding with this stage.

## WSL Architecture Operations

Deploy and maintain Linux distributions within the Windows host parameters. The fundamental host preparation must be completed prior to initialization.

* `wsl -l -v` : Audit installed distributions and their running versions.
* `wsl -l -o` : List distributions available for immediate deployment.

## Deployment and Upgrades

Initialize the core environment using the built-in deployment parameters.

* `wsl --install` : Deploy the default Ubuntu LTS distribution.
* `wsl --install -d <DistroName>` : Install a specific target distribution.
* `wsl --update` : Fetch and apply the latest WSL kernel updates.
