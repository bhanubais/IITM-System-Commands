# 03.03 Archival and Compression

## Purpose

Package file trees.

Compress payloads for backup, transfer, or storage.

## Strict Syntax

```bash
tar -cf archive.tar <path>
tar -czf archive.tar.gz <path>
tar -xf archive.tar
tar -xzf archive.tar.gz
tar -tf archive.tar.gz
gzip <file>
gunzip <file.gz>
zip archive.zip <files>
unzip archive.zip
```

## Command Map

| Command | Use |
|---|---|
| `tar -c` | Create archive. |
| `tar -x` | Extract archive. |
| `tar -t` | List archive contents. |
| `tar -f` | Specify archive file. |
| `tar -z` | Use gzip compression. |
| `gzip` | Compress one file and replace original. |
| `gunzip` | Decompress `.gz` file. |
| `xz` | High-density compression. |
| `bzip2` | Alternate compression. |
| `zip` | Cross-platform archive format. |
| `unzip` | Extract zip archive. |

## Atomic Examples

```bash
tar -cf logs.tar logs/                # create uncompressed archive
tar -czf logs.tar.gz logs/            # create gzip-compressed archive
tar -tf logs.tar.gz                   # list contents without extracting
tar -xf logs.tar                      # extract uncompressed archive
tar -xzf logs.tar.gz                  # extract gzip-compressed archive
tar -xzf logs.tar.gz -C /tmp/restore  # extract under target directory
gzip access.log                       # replaces file with access.log.gz
gunzip access.log.gz                  # restores file and removes .gz
xz archive.tar                        # higher compression, more CPU
bzip2 archive.tar                     # alternate compressor
zip reports.zip report1.txt report2.txt # create cross-platform archive
unzip reports.zip
du -h logs.tar.gz
```

## Compound Scenarios

Create a rollback archive before a release:

```bash
tar -czf site_backup_01.tar.gz /var/www/html
du -h site_backup_01.tar.gz
tar -tf site_backup_01.tar.gz | head
```

Restore archive into a target directory:

```bash
mkdir -p /tmp/restore
tar -xzf site_backup_01.tar.gz -C /tmp/restore
```

Package reports for cross-platform sharing:

```bash
zip reports.zip *.txt
unzip reports.zip -d reports-check
```

## Exam Traps

`tar -f` must be followed by the archive filename.

`tar -t` lists without extraction.

`gzip file` replaces `file` with `file.gz`.

Extracting archives can overwrite files. Inspect with `tar -tf` first.

## Production Notes

List archive contents before restore.

Extract into a staging directory when payload paths are uncertain.

Use `tar.gz` for Linux backups and `zip` for multi-OS handoff.
