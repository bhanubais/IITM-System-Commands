# 06.01 Shell Configurations and Prompts

## Purpose

Configure shell startup behavior, prompt content, command-line editing mode, and history behavior.

The shell prompt is an operational safety surface. It should show where commands will run.

## Strict Syntax

```bash
source ~/.bashrc
. ~/.bashrc
set -o vi
set -o emacs
export PS1='\u@\h:\w\$ '
history
```

## Command Map

| Item | Use |
|---|---|
| `.bash_profile` | Login shell startup file. |
| `.bash_login` | Alternate login startup file. |
| `.profile` | Portable login startup file. |
| `.bashrc` | Interactive non-login Bash startup file. |
| `.bash_logout` | Login shell exit file. |
| `PS1` | Primary prompt. |
| `PS2` | Continuation prompt. |
| `PS3` | `select` prompt. |
| `PS4` | Debug trace prompt. |
| `set -o vi` | Use vi-style command-line editing. |
| `set -o emacs` | Use Emacs-style command-line editing. |

## Atomic Examples

```bash
printf '%s\n' "$PS1"
export PS1='\u@\h:\w\$ '
export PS1='\u@\H:\w\$ '
set -o vi
set -o emacs
source ~/.bashrc
. ~/.bashrc
history
history 20
printf '%s\n' "$HISTSIZE"
printf '%s\n' "$HISTFILESIZE"
```

## Prompt Escapes

| Escape | Meaning |
|---|---|
| `\u` | Current username. |
| `\h` | Hostname before first period. |
| `\H` | Fully qualified hostname. |
| `\w` | Current working directory. |
| `\W` | Basename of current directory. |
| `\d` | Date. |
| `\t` | 24-hour time. |
| `\$` | `#` for root, `$` otherwise. |

## Compound Scenarios

Persist vi command-line editing:

```bash
printf '%s\n' 'set -o vi' >> ~/.bashrc
source ~/.bashrc
```

Set a context-rich prompt:

```bash
export PS1='\u@\H:\w\$ '
```

Append history after each prompt for multi-terminal work:

```bash
export PROMPT_COMMAND='history -a'
```

## Exam Traps

Login shells and interactive shells read different startup files.

`PS1` is the normal prompt. `PS2` appears when a command is incomplete.

`source file` and `. file` execute in the current shell.

## Production Notes

Prompts should expose user, host, and working directory.

Do not put secrets into shell history.

Use `read -s` for secret input and `set +o history` when history must be disabled temporarily.
