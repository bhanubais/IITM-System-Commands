# 05.04 Sed Stream Editor

## Purpose

Edit streams line by line.

Use `sed` for substitution, deletion, selective printing, and scripted text rewrites.

## Strict Syntax

```bash
sed 's/old/new/' <file>
sed -e 'script' <file>
sed -n -e 'addressp' <file>
sed -E -e 'extended_regex_script' <file>
sed -f <script_file> <file>
```

## Command Map

| Command | Use |
|---|---|
| `p` | Print pattern space. |
| `d` | Delete pattern space. |
| `s/old/new/` | Substitute first match. |
| `s/old/new/g` | Substitute all matches on the line. |
| `=` | Print line number. |
| `w file` | Write matching output to file. |
| `-n` | Suppress automatic printing. |
| `-e` | Provide inline script. |
| `-E` | Use Extended Regular Expressions. |
| `-f` | Read script from file. |

## Atomic Examples

```bash
sed 's/192.168.1.5/10.0.0.9/g' app_config.yml # replace all matches per line
sed -e 's/old/new/g' file.txt       # inline script with global substitution
sed -e '3,$d' file.txt              # delete from line 3 through end
sed -e '/^#/d' -e '/^$/d' file.txt  # delete comments and empty lines
sed -n -e '5p' sample.txt           # print only line 5
sed -n -e '5!p' sample.txt          # print all except line 5
sed -n -e '5,8p' sample.txt         # print line range
sed -n -e '5,8{=;p}' sample.txt     # print line numbers and lines in range
sed -n -e '1~2p' sample.txt         # GNU sed: every other line from line 1
sed -n -e '/microsoft/p' sample.txt # print matching lines
sed -n -e '/adobe/,+2p' sample.txt  # match plus two following lines
sed -f clean.sed input.txt          # run sed script file
```

## Compound Scenarios

Dry run a configuration migration:

```bash
sed 's/192.168.1.5/10.0.0.9/g' app_config.yml
```

Strip comments and empty lines:

```bash
sed -e '/^#/d' -e '/^$/d' app.conf
```

Print a range with line numbers:

```bash
sed -n -e '5,8{=;p}' sample.txt
```

## Exam Traps

Without `-n`, `sed` prints pattern space automatically.

Use `-n` with `p` when you need precise output.

Test scripts before using in-place editing.

Single quotes protect sed scripts from shell interpolation.

## Production Notes

Run a dry output check before editing files in place.

Use multiple `-e` scripts for readable pipelines.

Use `sed -f` when transformations become repeatable procedures.
