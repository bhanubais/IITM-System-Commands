# 06.03 Tmux Multiplexing

Engineer concurrent shell operations within a unified graphical interface leveraging `tmux`. The structural root contains multiple windows. Each window supports independent execution panes.

## Real World Scenario: Log Triangulation
During a critical software deployment, a systems engineer must monitor three distinct processes simultaneously. Instead of opening three separate terminal applications that might overlap or get lost, they launch `tmux`. They split the unified graphical window into three panes: Pane 1 streams CPU metrics via `top`, Pane 2 runs the live deployment script, and Pane 3 runs an active `tail -f` on the application error log. The entire multiplexed session can be detached and safely reattached later seamlessly.

## System Controls
Operations require the primary execution trigger sequence `Ctrl`+`B`.

* `Ctrl`+`B` followed by `?` : Display the internal documentation array. Strike `q` to exit.
* `Ctrl`+`B` followed by `c` : Construct a completely new operational window.
* `Ctrl`+`B` followed by `[0-9]` : Jump directly to the specified window integer.
* `Ctrl`+`B` followed by `n` : Cycle cleanly to the next numerical window.
* `Ctrl`+`B` followed by `p` : Cycle cleanly to the previous numerical window.
* `Ctrl`+`B` followed by `l` : Target the most recently engaged active window.
