# 04.01 vi Operations

## Purpose

Edit files on remote systems where graphical editors are unavailable.

`vi` requires mode awareness: command mode, insert mode, and ex mode.

## Strict Syntax

```bash
vi <file>
sudo vi <file>
```

Inside `vi`:

```text
i
Esc
:w
:q!
:wq
/pattern
:n1,n2s/old/new/g
```

## Command Map

| Key | Use |
|---|---|
| `Esc` | Return to command mode. |
| `i`, `I` | Insert at cursor or line start. |
| `a`, `A` | Append after cursor or line end. |
| `o`, `O` | Open line below or above. |
| `h`, `j`, `k`, `l` | Move left, down, up, right. |
| `0`, `$` | Move to line start or end. |
| `w`, `b` | Move by words. |
| `G`, `1G`, `nG` | End, start, or line `n`. |
| `x`, `dw`, `dd` | Delete character, word, or line. |
| `yy`, `p` | Copy line and paste. |
| `u` | Undo. |
| `:w`, `:q!`, `:wq` | Write, quit without saving, write and quit. |

## Atomic Examples

```text
i
A
o
Esc
dd
5dd
yy
p
/Port 22
n
:set nu
:set nonu
:1,5s/line/LINE/g
:w

:wq
```

## Compound Scenarios

Patch SSH config over a terminal:

```bash
sudo vi /etc/ssh/sshd_config
```

Inside `vi`:

```text
/Port 22
i
Esc
:wq
```

Jump to a compiler error line:

```text
:set nu
42G
```

Replace text across a range:

```text
:10,20s/old/new/g
:w
```

## Exam Traps

Typing text in command mode runs commands.

Use `Esc` to recover mode state.

`:q!` discards unsaved changes.

`dd` deletes a line. `5dd` deletes five lines.

## Production Notes

Use `sudo vi` only when the target file requires privilege.

Search first, edit second, write last.

Enable line numbers when matching logs or compiler output.
