# Toggle among multiple directories

Navigating a complex filesystem manually introduces unnecessary friction. You already know that the `cd -` command handles simple transitions between two locations. Managing multiple active paths simultaneously demands a more systematic approach. We will implement the Bash directory stack to automate these transitions and eliminate repetitive path entry.

Bash provides the directory stack natively. It tracks visited locations in a local memory queue. This is highly effective when working across a fixed set of project directories.

Consider a web development environment requiring constant movement between four distinct paths:
- `~/website/etc/apache2`
- `~/website/src`
- `~/website/ssl/certs`
- `~/website/www/html`

We will manage these transitions using native shell built-ins.

## The Directory Stack Data Structure

The directory stack is a Last-In, First-Out (LIFO) queue maintained directly by the shell. It records your path history. You manipulate the stack using three primary commands. The top of the stack is always the current working directory.

| Command | Execution Action | Stack Modification |
| :--- | :--- | :--- |
| **`dirs -v`** | Prints the current stack with zero-based index values. | None. Index `0` remains the current path. |
| **`pushd <path>`** | Changes the working directory to `<path>`. | Pushes `<path>` to index `0`. Shifts all existing entries down by one index. |
| **`popd`** | Changes the working directory to the new top entry. | Removes the top entry (index `0`). |
| **`pushd`** | Changes the working directory to index `1`. | Swaps index `0` and index `1`. Fails if the stack has fewer than two items. |
| **`pushd +N`** | Changes the working directory to the new top entry. | Rotates the stack left by `N` positions. Index `N` becomes index `0`. |
| **`popd +N`** | Maintains the current working directory. | Removes the specific entry at index `N`. |

## Stack Execution Workflow

The following sequence demonstrates stack initialization, path rotation, and entry removal. We will execute `dirs -v` after every modification to verify the exact state of the stack.

```bash
# Verify the starting location
❯ pwd
~/website

# The stack initializes with the current directory at index 0
❯ dirs -v
 0  ~/website

# Push new directories onto the stack
❯ pushd etc/apache2/
❯ pushd ~/website/src
❯ pushd ~/website/ssl/certs/
❯ pushd ~/website/www/html/

# Verify the populated stack state
❯ dirs -v
 0  ~/website/www/html
 1  ~/website/ssl/certs
 2  ~/website/src
 3  ~/website/etc/apache2
 4  ~/website

# Swap the top two directories (identical to `cd -`)
❯ pushd
~/website/ssl/certs ~/website/www/html ~/website/src ~/website/etc/apache2 ~/website

❯ dirs -v
 0  ~/website/ssl/certs
 1  ~/website/www/html
 2  ~/website/src
 3  ~/website/etc/apache2
 4  ~/website

# Rotate the stack to bring the original base directory to the top
❯ pushd +4
~/website ~/website/ssl/certs ~/website/www/html ~/website/src ~/website/etc/apache2

❯ dirs -v
 0  ~/website
 1  ~/website/ssl/certs
 2  ~/website/www/html
 3  ~/website/src
 4  ~/website/etc/apache2

# Rotate the stack to jump directly to the html directory
❯ pushd +2
~/website/www/html ~/website/src ~/website/etc/apache2 ~/website ~/website/ssl/certs

❯ dirs -v
 0  ~/website/www/html
 1  ~/website/src
 2  ~/website/etc/apache2
 3  ~/website
 4  ~/website/ssl/certs

# Remove the current top entry and drop into the source directory
❯ popd
~/website/src ~/website/etc/apache2 ~/website ~/website/ssl/certs

❯ dirs -v
 0  ~/website/src
 1  ~/website/etc/apache2
 2  ~/website
 3  ~/website/ssl/certs

# Remove a specific background entry without leaving the current directory
❯ popd +2
~/website/src ~/website/etc/apache2 ~/website/ssl/certs

❯ dirs -v
 0  ~/website/src
 1  ~/website/etc/apache2
 2  ~/website/ssl/certs
```
