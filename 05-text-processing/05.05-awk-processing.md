# 05.05 AWK Data Processing

## Purpose

Process field-oriented text with conditions, actions, and small programs.

Use `awk` when simple filters become record logic.

## Strict Syntax

```bash
awk 'pattern { action }' <file>
awk -F '<delimiter>' '{ print $1 }' <file>
awk -f <script.awk> <file>
```

## Command Map

| Construct | Use |
|---|---|
| `$0` | Whole current record. |
| `$1`, `$2` | First and second fields. |
| `-F` | Input field separator. |
| `NR` | Total record number. |
| `FNR` | Record number in current file. |
| `BEGIN` | Run before input. |
| `END` | Run after input. |
| `-f` | Load program file. |

## Atomic Examples

```bash
awk '{print $1}' file.txt
awk -F ':' '{print $1}' /etc/passwd  # colon field separator
awk -F ':' '$7 == "/bin/bash" {print $1}' /etc/passwd # condition on login shell
awk '{print NR, $0}' file.txt         # print record number and whole line
awk 'END {print NR}' file.txt         # total line count after input
awk -F ',' '{print $1, $3}' data.csv  # comma separator, fields 1 and 3
awk -f script.awk input.txt           # run program file
awk -f lib.awk -f main.awk input.txt  # load library before main program
```

## Compound Scenarios

List users with Bash login shells:

```bash
awk -F ':' '$7 == "/bin/bash" {print $1}' /etc/passwd
```

Count records per first field:

```bash
awk '{count[$1]++} END {for (k in count) print count[k], k}' access.log | sort -nr # count by first field
```

Run a reusable AWK program:

```bash
chmod +x myscript.awk
./myscript.awk input.txt
```

## Exam Traps

Default field separator is runs of whitespace.

`-F ':'` changes input splitting, not output formatting.

`$0` is the full record. `$1` is the first field.

`gawk` extensions are not always POSIX `awk`.

## Production Notes

Pass filenames to `awk` rather than piping large files unnecessarily.

Use `-f` when logic needs versioned review.

Keep field separators explicit for system files such as `/etc/passwd`.
