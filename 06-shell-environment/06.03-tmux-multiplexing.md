# 06.03 Tmux Multiplexing

## Purpose

Keep terminal work alive across disconnects.

Run multiple shell panes inside one session.

## Strict Syntax

```bash
tmux
tmux new -s <name>
tmux ls
tmux attach -t <name>
tmux detach
```

Inside tmux, the default prefix is `Ctrl+B`.

## Command Map

| Command or Key | Use |
|---|---|
| `tmux` | Start unnamed session. |
| `tmux new -s name` | Start named session. |
| `tmux ls` | List sessions. |
| `tmux attach -t name` | Attach to session. |
| `Ctrl+B ?` | Help. |
| `Ctrl+B c` | New window. |
| `Ctrl+B 0..9` | Jump to numbered window. |
| `Ctrl+B n` | Next window. |
| `Ctrl+B p` | Previous window. |
| `Ctrl+B l` | Last active window. |

## Atomic Examples

```bash
tmux
tmux new -s deploy
tmux ls
tmux attach -t deploy
```

```text
Ctrl+B ?
Ctrl+B c
Ctrl+B 0
Ctrl+B n
Ctrl+B p
Ctrl+B l
```

## Compound Scenarios

Start a deployment monitoring session:

```bash
tmux new -s deploy
```

Inside tmux:

```text
Ctrl+B c
```

Recover after reconnecting:

```bash
tmux ls
tmux attach -t deploy
```

Use separate windows for logs and commands:

```text
Ctrl+B c
Ctrl+B n
Ctrl+B p
```

## Exam Traps

The prefix key is required before tmux control keys.

Detaching a tmux session is not the same as ending the processes inside it.

Named sessions are easier to recover than unnamed sessions.

## Production Notes

Use `tmux` before long SSH work.

Name sessions by task, such as `deploy`, `backup`, or `incident`.

Run logs, deployment commands, and monitors in separate windows.
