# 02.01 Directory Governance

## Purpose

Move through the filesystem and create or inspect directory nodes.

This is the base layer for every file operation that follows.

## Strict Syntax

```bash
pwd
cd <path>
mkdir <directory>
mkdir -p <path>
rmdir <directory>
ls [options] [path]
tree [options] [path]
```

## Command Map

| Command | Use |
|---|---|
| `pwd` | Print current working directory. |
| `cd` | Change directory. |
| `mkdir` | Create one directory. |
| `mkdir -p` | Create parent directories as needed. |
| `rmdir` | Remove an empty directory. |
| `ls` | List directory contents. |
| `ls -ld` | Inspect the directory node itself. |
| `tree` | Display hierarchy as a tree. |

## Atomic Examples

```bash
pwd
echo "$PWD"
cd /usr/bin
cd ..
cd ~
cd /
mkdir logs
mkdir -p app/cache/tmp
rmdir empty-dir
ls
ls -a
ls -l
ls -ld /tmp
tree
tree -a
tree -d
```

## Compound Scenarios

Create a project directory tree and verify it:

```bash
mkdir -p project/{bin,config,logs}
ls -ld project project/bin project/config project/logs
```

Audit a directory without listing all children:

```bash
cd /var
pwd
ls -ld log
```

Map only directories in a workspace:

```bash
tree -d .
```

## Exam Traps

The parent of `/` is `/`.

Extra forward slashes are normalized in paths, so `////usr////bin` resolves as `/usr/bin`.

`rmdir` removes only empty directories.

`ls -l dir` lists contents. `ls -ld dir` lists the directory node.

## Production Notes

Print `pwd` before destructive commands.

Use `mkdir -p` in scripts when parent directories may not exist.

Use `ls -ld` before changing ownership or permissions on a directory.
