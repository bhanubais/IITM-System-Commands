# 02.03 Link Architecture (Hard and Soft)

Disks manage stored file and folder data in structures that assign a unique ID known as an `inode`. A file or folder is accessed through instances of its path or its absolute `inode`.

You can create multiple access points for a single file. Utilizing the `inode` directly generates a hard link. Utilizing the descriptive path generates a soft link.

### Link Operational Parameters
* `ln -i` : Interactive mode. Halt and request authorization before overwriting target linkage.
* `ln -b` : Generate automated backup copies of targets prior to link association (designated with a `~` suffix).
* `ln -d` : Override restriction and allow hard-linking of directories. Required explicitly to be processed by superuser via `sudo`.

## Soft Links (Symbolic Links)
A soft link operates as a pointer to an absolute or relative path. If the target file is relocated or deleted, the soft link fractures and fails.

* `ln -s /path/to/target/file path/to/symlink` : Construct a soft link pointing to the defined file or directory.
* `ln -s --force /path/to/new_target path/to/symlink` : Overwrite an active symbolic link to redirect it toward a new target.

## Hard Links
A hard link operates as a direct reference to the fundamental `inode` representing data on a specific disk. They are fundamentally restricted to operating exclusively within the boundaries of the identical physical device as the source.

Moving or deleting the original user-facing file does not destroy the data or fracture the link. The data persists until all hard links pointing to that `inode` are purged.

* `ln /path/to/target/file path/to/hardlink` : Construct a hard link directly to the target file `inode`.

## Architectural Link Breakage Physics
1. Assume `file_1` exists within `inner_folder/`.
2. Generate hard link: `ln inner_folder/file_1 file_1_hard`.
3. Generate soft link: `ln -s inner_folder/file_1 file_2_soft`.
4. Result:
   - If `inner_folder/file_1` is relocated to a new node, `file_2_soft` fractures and outputs a broken path reading error. `file_1_hard` continues to function.
   - If `inner_folder/file_1` is permanently deleted, `file_2_soft` fails completely. `file_1_hard` retains the complete uncorrupted dataset natively attached to the fundamental memory sector.

Directories possess a structural constraint. Standard execution necessitates engineering soft links exclusively for directories. Generating hard links for directories initiates infinite loop hazards through cross-referencing recursion.
