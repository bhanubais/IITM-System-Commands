# 03.01 Debian Software Governance

## Purpose

Install, update, remove, and audit Debian packages.

Use `apt` and `apt-get` for repository-aware dependency resolution. Use `dpkg` for local package files and installed-package inspection.

## Strict Syntax

```bash
sudo apt-get update
sudo apt-get install <package>
sudo apt-get remove <package>
sudo apt-get purge <package>
sudo apt autoremove
dpkg -l <pattern>
dpkg -L <package>
dpkg -S <path>
sudo dpkg -i <file.deb>
```

## Command Map

| Command | Use |
|---|---|
| `apt-cache search` | Search repository package names and descriptions. |
| `apt-cache show -a` | Show package metadata. |
| `apt-get update` | Refresh local package indexes. |
| `apt-get install` | Install package and dependencies. |
| `apt-get upgrade` | Upgrade installed packages. |
| `apt-get remove` | Remove package binaries, preserve config. |
| `apt-get purge` | Remove package and config. |
| `apt autoremove` | Remove unused dependencies. |
| `dpkg -l` | List installed packages by pattern. |
| `dpkg -L` | List files installed by package. |
| `dpkg -S` | Find package owning a path. |
| `dpkg-query` | Format package database output. |
| `dpkg -i` | Install local `.deb` file. |

## Atomic Examples

```bash
apt-cache search apache2
apt-cache show -a apache2             # inspect all available package records
sudo apt-get update                   # refresh local package index
sudo apt-get install apache2
sudo apt-get upgrade                  # upgrade installed packages
sudo apt-get remove apache2           # remove binary, keep config
sudo apt-get purge apache2            # remove binary and config
sudo apt autoremove                   # remove unused dependencies
dpkg -l '*nginx*'                     # list installed packages matching pattern
dpkg -L openssh-server                # list paths installed by package
dpkg -S /etc/ssh/sshd_config          # find package owning a path
sudo dpkg -i package.deb              # install local package file
sudo apt --fix-broken install         # repair missing dependencies
dpkg-query -W -f='${Package} ${Version}\n' # custom package query output
```

## Compound Scenarios

Install a web server with index refresh:

```bash
apt-cache search apache2
sudo apt-get update
sudo apt-get install apache2
```

Audit installed package sections:

```bash
dpkg-query -W -f='${Section} ${binary:Package}\n' | sort | uniq -c | sort -rn
```

Trace package changes:

```bash
cat /var/log/dpkg.log
sudo tail -n 100 /var/log/auth.log
```

## Exam Traps

`apt-get update` updates package indexes. It does not upgrade installed packages.

`remove` preserves configuration files.

`purge` removes package configuration files.

`dpkg -i` does not resolve remote dependencies by itself.

## Production Notes

Run `apt-get update` before install when indexes may be stale.

Prefer repository packages for system daemons and core tools.

Use containers or snaps only when isolation or version constraints justify the extra layer.
