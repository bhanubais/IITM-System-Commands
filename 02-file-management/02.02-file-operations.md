# 02.02 File Operations

Execute direct manipulation of file entities. Engineer file creation, relocation, and deletion protocols.

## Executing Identity
File modification rights depend strictly on group association limits.
* `groups` : Output all structural groups associated with the active user.

## State Modifications

### Creation and Timestamps (touch)
* `touch <file_name>` : Initialize an empty file node. If the file exists, it calibrates the modification timestamp to the present moment without data destruction.
* `touch --no-create <file>` : Update timestamps on existing files without initializing new ones if missing.
* `touch -t YYYYMMDDHHMM.SS <file>` : Inject a specific absolute timestamp.
* `touch --reference=<source> <target>` : Calibrate the target file timestamp to exactly match the source file.

### Duplication (cp)
* `cp <source> <target>` : Duplicate file data from a source node to a target node.
* `cp -f <source> <target>` : Force overwrite. Execute destructive writing if the target already exists. Behavior: `echo "content" > f1; cp f1 f2` permanently destroys the prior content of `f2`.
* `cp -p <source> <target>` : Preserve mode, ownership, and timestamp parameters during the duplication.
* `cp --parents <source> <target>` : Duplicate the file while retaining absolute source directory structure under the target.
* `cp --target-directory=<dir> <file1> <file2>` : Explicitly route multiple files into a defined target node.

### Relocation (mv)
* `mv <source> <target>` : Relocate a file node. This is identical to renaming when the target directory remains unchanged.

## Deletion Protocols
Data removal is permanent. Execute strict oversight.
* `rm <file_name>` : Purge a specific file.
* `rm -r <dir_name>` : Recursively purge a directory and all descendant nodes.
* `rm -ri <dir_name>` : Recursively purge a directory but halt for interactive authorization prior to every deletion.
* `rm *` : Purge all unstructured files within the current active directory.
 Example: `cp *.sh backup_dir/` routes all shell scripts to the backup node.
* `?` : Match any single character exactly once.
* `[set]` : Match any single character defined within the brackets. Example: `[A-Z]`.
* `[^set]` or `[!set]` : Inverse match. Target any single character explicitly absent from the set. Example: `rm [!a-z]*` deletes files starting with non-lowercase letters
* `*` : Match zero or more consecutive characters.

* `\` : Escape the subsequent character to bypass pattern interpretation.

## Brace Expansion
Automate the generation of multiple arguments prior to command execution.

* `echo file{1..3}.txt` : Expands to `file1.txt file2.txt file3.txt`.
* `echo {A..E}` : Expands to a linear sequence `A B C D E`.
* `echo {fast,slow}-server.conf` : Expands into a comma-separated permutation `fast-server.conf slow-server.conf`.
