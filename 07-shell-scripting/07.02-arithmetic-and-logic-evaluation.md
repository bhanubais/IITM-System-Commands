# 07.02 Arithmetic and Logic Evaluation

## Purpose

Evaluate strings, integers, files, exit status, and arithmetic expressions in shell scripts.

## Strict Syntax

```bash
[ expression ]
[[ expression ]]
(( arithmetic ))
$(( arithmetic ))
expr expression
echo 'scale=2; expression' | bc -l
```

## Command Map

| Syntax | Use |
|---|---|
| `[ ]` | POSIX test expression. |
| `[[ ]]` | Bash conditional expression. |
| `(( ))` | Arithmetic evaluation. |
| `$(( ))` | Arithmetic expansion. |
| `expr` | Legacy expression evaluation. |
| `bc -l` | Floating point calculation. |
| `$?` | Previous command exit status. |

## Atomic Examples

```bash
[[ "A" = "B" ]]
[[ "A" != "B" ]]
[[ -v var ]]                         # Bash: true if variable is set
[ -z "$VAR" ]                        # true if empty or unset
[ -n "$VAR" ]                        # true if non-empty
[[ -z ${VAR+x} ]]                    # true only if unset
[ "$n" -eq 3 ]                       # integer equal
[ "$n" -ne 3 ]                       # integer not equal
[ "$n" -gt 3 ]                       # integer greater than
[ "$n" -ge 3 ]                       # integer greater or equal
[ "$n" -lt 3 ]                       # integer less than
[ "$n" -le 3 ]                       # integer less or equal
[ -e file.txt ]                      # path exists
[ -f file.txt ]                      # regular file
[ -d logs ]                          # directory
[ -s app.log ]                       # exists and non-empty
[ -r file.txt ]                      # readable by current user
[ -w file.txt ]                      # writable by current user
[ -x script.sh ]                     # executable or traversable
echo $((4 + 5))                      # arithmetic expansion
echo $((4 == 4))                     # arithmetic truth: 1 true, 0 false
let a=5+4                            # assign arithmetic result
expr 5 + 4
expr 5 \* 4                          # escape * from shell globbing
echo '4.3 + 2.3' | bc -l             # floating point via bc math library
```

## Compound Scenarios

Rotate a log only when it is non-empty:

```bash
if [ -s app.log ]; then
	gzip app.log
fi
```

Validate a required variable:

```bash
if [ -n "$MY_VAR" ]; then
	printf 'set\n'
else
	printf 'empty or unset\n' >&2
fi
```

Combine integer checks:

```bash
if [[ $count -gt 10 && $count -lt 100 ]]; then
	printf 'inside range\n'
fi
```

## Exam Traps

String equality uses `=` inside `[ ]` and `[[ ]]`.

Integer comparison uses `-eq`, `-ne`, `-gt`, `-ge`, `-lt`, and `-le`.

`[ -v VAR ]` is not POSIX and is not valid in all shells.

Bash integer arithmetic does not handle floating point.

## Production Notes

Quote variables inside `[ ]`.

Use `[[ ]]` for Bash scripts when pattern and logical expressions need cleaner syntax.

Use `bc -l` for floating point math.
