# 07.06 Parameter Expansion and Quoting

## Purpose

Control how shell values expand.

This file covers defaults, slicing, pattern removal, substitution, case conversion, arrays through expansion syntax, and quoting.

## Strict Syntax

```bash
${parameter}
${parameter:-word}
${parameter:=word}
${parameter:+word}
${parameter:offset:length}
${parameter#pattern}
${parameter%pattern}
"$parameter"
'literal text'
```

## Command Map

| Expansion | Meaning |
|---|---|
| `${#var}` | Length of `var`. |
| `${var:-default}` | Use default if unset or null. |
| `${var:=default}` | Assign default if unset or null. |
| `${var:+value}` | Use value only if variable is set and not null. |
| `${var:5:4}` | Extract four characters after offset five. |
| `${var#pattern}` | Remove shortest front match. |
| `${var##pattern}` | Remove longest front match. |
| `${var%pattern}` | Remove shortest rear match. |
| `${var%%pattern}` | Remove longest rear match. |
| `${var/pattern/new}` | Replace first match. |
| `${var//pattern/new}` | Replace all matches. |
| `${var^^}` | Convert to uppercase in Bash. |
| `${var,,}` | Convert to lowercase in Bash. |

## Atomic Examples

```bash
file="/var/log/nginx/access.log"
printf '%s\n' "${#file}"
printf '%s\n' "${file##*/}"
printf '%s\n' "${file%/*}"
printf '%s\n' "${file%.log}"
printf '%s\n' "${file/access/error}"
printf '%s\n' "${file//\//_}"
printf '%s\n' "${target:-/etc/default/app}"
printf '%s\n' "${target:=/etc/default/app}"
printf '%s\n' "${target:+configured}"
name="prod-api"
printf '%s\n' "${name:0:4}"
printf '%s\n' "${name^^}"
printf '%s\n' "${name,,}"
```

## Quoting Rules

Single quotes preserve every character literally:

```bash
printf '%s\n' 'Hello $USER'
```

Double quotes allow parameter and command expansion while blocking word splitting and glob expansion:

```bash
printf '%s\n' "Hello $USER"
```

Backslash escapes one following character outside single quotes:

```bash
printf '%s\n' "HOME is \$HOME"
```

Embedding a single quote inside single-quoted text requires closing and reopening:

```bash
printf '%s\n' 'It'"'"'s done'
```

## Compound Scenarios

Derive service name, directory, and extension from a path:

```bash
path="/srv/api/releases/app.tar.gz"
dir="${path%/*}"
base="${path##*/}"
name="${base%%.*}"
printf 'dir=%s base=%s name=%s\n' "$dir" "$base" "$name"
```

Set defaults without losing explicit configuration:

```bash
log_dir="${LOG_DIR:-/var/log/app}"
retention_days="${RETENTION_DAYS:-7}"
printf 'log_dir=%s retention=%s\n' "$log_dir" "$retention_days"
```

Normalize a deployment label:

```bash
raw="Prod API 01"
label="${raw// /-}"
printf '%s\n' "${label,,}"
```

## Exam Traps

`${var:-word}` does not assign.

`${var:=word}` assigns.

`${var#pattern}` and `${var%pattern}` remove from opposite sides.

Unquoted `$var` can split into multiple words and expand globs.

## Production Notes

Quote scalar expansions by default.

Use `${var:?message}` in critical scripts when a missing value must abort execution.

Prefer parameter expansion over external tools for simple basename, dirname, and replacement logic.
