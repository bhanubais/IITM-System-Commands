# 03.02 Checksum Validation

Checksum generation is a mandatory checkpoint. Do not deploy packages or raw data streams without verifying source authenticity.

A microscopic mutation in the source block engineers a massive deviation in the generated checksum block. This discrepancy indicates corruption or malicious intrusion.

## Security Algorithms

* `md5sum <filename>` : Deploy the 128-bit cryptographic hashing function and output the verification string.
* `sha1sum <filename>` : Execute the 160-bit secure hash algorithm formulation.
* `sha256sum <filename>` : Standardize validation by invoking the 256-bit secure hash architecture. This is preferred for network data verification.

Ensure target output matches the authority string provided by the repository server exactly. Any variation represents critical verification failure.
