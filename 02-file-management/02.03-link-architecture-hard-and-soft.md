# 02.03 Link Architecture (Hard and Soft)

Disks manage stored file and folder data in structures that assign a unique ID known as an `inode`. A file or folder is accessed through instances of its path or its absolute `inode`.

You can create multiple access points for a single file. Utilizing the `inode` directly generates a hard link. Utilizing the descriptive path generates a soft link.

## Soft Links (Symbolic Links)
A soft link operates as a pointer to an absolute or relative path. If the target file is relocated or deleted, the soft link fractures and fails.

* `ln -s /path/to/target/file path/to/symlink` : Construct a soft link pointing to the defined file or directory.
* `ln -s --force /path/to/new_target path/to/symlink` : Overwrite an active symbolic link to redirect it toward a new target.

## Hard Links
A hard link operates as a direct reference to the fundamental `inode`. Moving or deleting the original user-facing file does not destroy the data or fracture the link. The data persists until all hard links pointing to that `inode` are purged.


* `ln /path/to/target/file path/to/hardlink` : Construct a hard link directly to the target file `inode`.

Directories possess a structural constraint. You are restricted to engineering soft links exclusively for directories. Generating hard links for directories initiates infinite loop hazards through cross-referencing recursion.
