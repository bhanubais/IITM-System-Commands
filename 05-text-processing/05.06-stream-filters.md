# 05.06 Stream Filters and Data Parsing

Execute deep string mutations and structural stream normalization. Engineer robust process pipelines by passing inputs through multiple isolated filtration binaries.

## Real World Scenario: Log Threat Sorting
You operate a cloud server under attack and possess a raw text file containing thousands of IP addresses hitting your firewall. You must extract exclusively the top 5 most aggressive unique attackers.
1. Target the file and alphabetize the data blocks so identical IPs group together: `sort attacker_ips.txt`
2. Pipe the block into `uniq -c` to count consecutive matches: `sort attacker_ips.txt | uniq -c`
3. Sort the block numerically and reversely to put the highest counts at the top: `sort attacker_ips.txt | uniq -c | sort -nr`
4. Use `head` to isolate just the primary 5 matrices: `sort attacker_ips.txt | uniq -c | sort -nr | head -n 5`

## Horizontal Column Extraction Operations (`cut`)
Trim text blocks along horizontal vectors parsing strict delimiters.

* `cut -c 1-4 <filename>` : Transmit character data exclusively bound between integer columns 1 and 4.
* `cut -d " " -f 1` : Establish spatial gaps `" "` as structural delimiters (`-d`). Export string segments populating the first field axis exclusively (`-f 1`).

## Metric Analytics (`wc`)
Quantify file limits and stream payloads structurally. Example pipeline context: `ls /bin | wc -l`.

* `wc -l` or `--lines` : Count distinct newline limits.
* `wc -w` or `--words` : Count distinct word segments.
* `wc -m` or `--chars` : Count distinct individual characters.
* `wc -c` or `--bytes` : Count byte footprint allocations explicitly.

## Data Block De-Duplication and Sorting
Sequentialize and isolate unique blocks.

* `sort <filename>` : Organize output lines alphabetically or by strictly defined integers.
* `sort -k 5,5 -t ' ' -nr` : Define structural delimiter mapping using the space character (`-t ' '`). Scale sorting recursively (`-r`) through integer arrays (`-n`), operating strictly on field segment index 5 (`-k 5,5`).
* `uniq` : Condense immediately consecutive identical memory segments into a single primary occurrence.
* `uniq -d` : Isolate matching clusters. Output duplicate blocks and drop unique sequence anomalies.
* `uniq -c` : Execute count iteration mapping consecutive matching structural blocks.

## Character Array Normalization (`tr`)
Execute translation protocols translating or discarding sequence elements directly via the pipe stream.

* `tr -s " "` : Compress all extensive gap segments spanning more than one uniform whitespace structural array into identical distinct spatial units.

## Stream Transformation Architectures
* `seq <n>` : Output integers parsing consecutively from 1 to `<n>`.
* `seq -w <n>` : Equalize width by padding numbers with leading zeros (useful for generating ordered filenames).
* `seq -s"<sep>" <n>` : Join sequence elements with a custom separator, e.g. `seq -s"-" 14`.
Examples:
```bash
# Padded filenames from 01 to 10
for i in $(seq -w 1 10); do
	printf 'file%s.txt\n' "$i"
done

# Single-line dash-separated sequence
seq -s "-" 1 14
```

## Column Joining with `paste`
Join parallel streams column-wise without writing temporary files. `paste` is simple, fast, and useful when you need side-by-side comparisons or to assemble report rows from separate sources.

Basic side-by-side join:
```bash
paste fileA.txt fileB.txt
```
Use a custom delimiter (comma, tab, or other):
```bash
paste -d ',' fileA.txt fileB.txt
```
Serial paste — concatenate lines from a single file into a single line using the given delimiter:
```bash
paste -s -d ' ' file.txt    # join all lines into one space-separated line
```
Work with streams and command outputs using process substitution:
```bash
paste <(cut -f1 fileA.tsv) <(cut -f2 fileB.tsv)
```
Practical example — assemble a two-column report from log fields:
```bash
cut -f1 access.log | sort | uniq -c > counts.txt
cut -f2 access.log > ips.txt
paste counts.txt ips.txt | sort -k1,1nr | head -n 10
```
Notes:
- If input files have different line counts, `paste` fills missing fields with empty strings. Use `pr` or align manually if padding is required.
- For scripts that must run on systems without process substitution, combine `paste` with temporary FIFOs (see the process-substitution doc for `mkfifo` fallback).

## Parsing CSV and delimited data
Use `IFS` with `read -r` in a `while` loop to parse delimited records safely. This pattern avoids word-splitting on fields and works well for simple CSV-like files.

Example — read a CSV line-by-line and split into columns:
```bash
while IFS=, read -r f1 f2 rest; do
	printf 'col1=%s col2=%s rest=%s\n' "$f1" "$f2" "$rest"
done < file.csv
```

Tips:
- Use `-r` to prevent backslash escapes from being interpreted.
- If fields may contain embedded commas, prefer a CSV-aware tool such as `csvtool`, `xsv`, `python -c`, or `awk -F,`.
- Skip a header line safely:
```bash
{ read -r header; while IFS=, read -r f1 f2 rest; do
	...
done; } < file.csv
```

## Extracting file size column from `ls -l`
`ls -l` output uses variable spacing that can break `cut`'s simple field extraction. Use `tr -s` to squeeze spaces, then `cut` to extract the size column and `sort` to rank by size.

Example:
```bash
# Show sizes only
ls -l | tr -s ' ' | cut -d ' ' -f5

# Sort by size (numeric, reverse)
ls -l | tr -s ' ' | sort -k5,5 -t ' ' -nr
```

Notes:
- `tr -s ' '` collapses repeated spaces into a single separator so `cut -d ' '` reliably addresses the fifth field.
- For robust scripting prefer `stat --printf='%s %n\n'` when available, or use `find -printf '%s %p\n'` for recursive size extraction.

## Splitting streams with `tee`
`tee` duplicates a stream: it writes `stdin` to one or more files and also to `stdout`. This is useful for saving intermediate results while continuing a pipeline.

Examples:
```bash
# Save output to a single file and continue the pipeline
ls $HOME | tee file1 | wc -l

# Save to multiple files
ls $HOME | tee file1 file2 | wc -l

# Write to the terminal and files (use /dev/tty)
egrep -l '^void[[:space:]]main\(\)[[:space:]]+{$' *.cpp | tee /dev/tty | wc -l

# Redirect stderr before tee to suppress error messages from the pipeline
ls $HOME /blah 2>/dev/null | tee file1 file2 | wc -l
```

Notes:
- `tee` overwrites files by default; use `tee -a` to append.
- When working with both stdout and stderr, place `2>` before `tee` if you want to suppress errors from reaching the files.
