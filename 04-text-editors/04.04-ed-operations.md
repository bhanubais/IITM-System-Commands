# 04.04 ed Operations

The `ed` utility acts as the original, standard UNIX line editor. Operational syntax adheres to the framework: `[address_1[,address_2]]command[parameters]`.

## Fundamental Execution State
* Enter `e <filename>` to load a file into the active buffer.
* Execute `w <filename>` to commit the buffer data to disk.
* Enter `q` to quit the editor.
* Trigger `P` to toggle the prompt status indicator.
* Execute `!command` to bypass the editor and run an external shell calculation.
* Read external command outputs directly into buffer with `r !command`.

## Editing Operations
* `p` : Print active target line to standard output.
* `n` : Display the target line prefixed with its integer line number.
* `i` : Insert a new data block exactly before the targeted line.
* `a` : Append a new data block exactly after the targeted line.
* `c` : Change the contents of the entire targeted line block.
* `d` : Delete the specific targeted line block.
* `m <pos>` : Move targeted line array to a new structural index.
* `u` : Revert the strict latest mutation.

All data insertions (`i`, `a`, `c`) must be aggressively terminated by placing a lone `.` on an empty line.

## Regex Substitution
The core search-replace structure parses as `s/query/replacement/g`.
Prefixing a line boundary confines the substitution range.
* `3,5s/PREFIX/prefix/` : Execute discrete substitution for the string "PREFIX" exclusively across structural lines 3 through 5.
