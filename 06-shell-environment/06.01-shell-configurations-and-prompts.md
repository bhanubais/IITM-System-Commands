# 06.01 Shell Configurations and Prompts

Initialize user environments using strict configuration files. Engineer shell prompts to output critical system data instantly.

## Real World Scenario: Secure Prompt Engineering
A systems engineer manages 50 different servers. To prevent deploying destructive commands (`rm -rf`) to a critical production database when they believe they are in a local development container, they engineer the `PS1` variable inside `~/.bashrc`. They set a logic parameter that turns the prompt bright red if the user is `root`, and includes the fully qualified hostname so the execution context is visibly absolute.

## Structural Shell Execution Sequence
Linux operates across a fundamental hierarchy ranging from the physical core to the graphical interface: Kernel → Programs → Shell → GUI. The Shell itself executes commands following a strict multi-layer sequential parser.

1. **Read**: Ingest the raw string.
2. **Break**: Tokenize the string by parsing spaces or delimiting operators.
3. **Parse**: Compile tokens into explicit operational parameters and flags.
4. **Expand**: Execute brace, tilde, parameter, variable, and glob expansion (e.g., resolving `*` or `$HOME`).
5. **Redirection**: Manage file descriptor routing (`>`, `<`).
6. **Execute**: Launch the target program or binary using the compiled system variables.
7. **Wait**: Suspend interactive capability until the execution yields an exit code.

## Startup and Initialization Files
The bash shell triggers specific initialization files during authentication and launch sequences.

* **Login Shells**: Execute `.bash_profile`, `.bash_login`, or `.profile` upon direct system authentication. Choose one configuration format strictly.
* **Interactive Shells**: Execute `.bashrc` upon launching a standard terminal window from a GUI.
* **Logout**: Execute `.bash_logout` explicitly when terminating the primary shell session.

## Prompt Infrastructure
Four primary prompt variables govern terminal context.

* `PS1` : Primary prompt string (default is `$`).
* `PS2` : Secondary prompt string for incomplete commands (default is `>`).
* `PS3` : Prompt string for `select` loop input in bash scripts.
* `PS4` : Execution trace prompt output when operating in debug mode via `set -x` (default is `+`).

## Escape Sequences
Calibrate `PS1` using backslash escape parameters.

* `\u` : Current active username.
* `\h` : Local hostname halting at the first period.
* `\H` : Absolute fully qualified hostname.
* `\w` : Absolute path of the current working directory.
* `\W` : Base structural name of the current directory.
* `\d` : Date structured as "weekday month day".
* `\t` : Strict 24-hour time clock formatted as hh:mm:ss.
* `\$` : Resolves to `#` if operating under root authority. Otherwise outputs `$`.

Example default Ubuntu `PS1` logic: `\u@\h:\w\$`.

## Shell Editing Modes
Bash supports two interactive command-line editing keymaps: Emacs and Vi. Configure the preferred style per-user in `~/.bashrc`.

- Enable Emacs mode (default):
```bash
set -o emacs
```
- Enable Vi mode:
```bash
set -o vi
```

Sample `.bashrc` snippet that sets Vi mode and enables a minimal visual cue:
```bash
# Use vi-style line editing
set -o vi

# Optionally show editing mode in PS1 when in interactive shells
if [[ $- == *i* ]]; then
	bind 'set show-mode-in-prompt on'
fi
```

Notes:
- `set -o vi` allows vi-style navigation and editing on the command line (use `Esc` to switch to command mode).
- `set -o emacs` restores Emacs-style keybindings.
- Add the chosen `set -o` line to `~/.bashrc` to persist behavior across sessions.

## Command history and recall
Efficient use of command history speeds interactive work and reduces repetition. Configure history behavior in `~/.bashrc` and use builtins to recall and edit commands.

Helpful commands:
```bash
history          # list recent commands
history N        # list last N commands
!!               # run previous command
!$               # expand to last argument of previous command
!N               # execute command number N from history
```

Configuration tips:
- Set `HISTSIZE` and `HISTFILESIZE` to control the in-memory and on-disk history length.
- `HISTCONTROL=ignoredups` avoids consecutive duplicate entries; `ignoreboth` combines `ignorespace` and `ignoredups`.
- Append to history immediately by adding `PROMPT_COMMAND='history -a'` in `~/.bashrc` for multi-terminal sessions.

Safety:
- Avoid storing sensitive data (passwords, tokens) in shell history. Use `read -s` for secret input and disable history temporarily with `set +o history` when necessary.
