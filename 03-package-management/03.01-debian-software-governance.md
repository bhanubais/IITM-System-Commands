# 03.01 Software Management (Debian/APT)

Execute structural control over external binaries using Advanced Package Tool (`apt`) and Debian Package management (`dpkg`). Software installation mutates system capabilities and requires absolute precision.

## Real World Scenario: Web Server Deployment
During a hackathon or a rapid cloud deployment, you must build a functional web server quickly while ensuring strict logging and dependency management.

1. `apt-cache search apache2` : Query the remote database to ensure the `apache2` package exists.
2. `sudo apt-get update` : Synchronize the local repository index before pulling code. Failure to run this may result in 404 Not Found errors during download.
3. `sudo apt-get install apache2` : Deploy the target package directly onto the operating system.

## Advanced Package Tool (APT) Operations
APT functions as the primary high-level package resolution framework. It automatically solves system dependency errors that arise during installation.

* `sudo apt-get upgrade` : Scale all active installed packages to their most recent secure versions.
* `apt-cache show -a <package_name>` : Extract granular package specification records, such as file size, maintainer contact, and checksum data.
* `sudo apt-get remove <package_name>` : Purge the package binary but preserve independent configuration files. Useful when reinstalling faulty software without losing custom settings.
* `sudo apt-get purge <package_name>` : Exterminate both the target package and its internal configurations completely.
* `sudo apt autoremove` : Audit and execute removal of all decoupled, obsolete dependency packages. This clears disk space heavily polluted by removed toolchains.

## Base Debian Package (dpkg)
For offline installation and granular local auditing, leverage `dpkg`. Do not utilize `dpkg` for network resolution, as it entirely bypasses external dependency graphs.

* `dpkg -l <pattern>` : List all locally installed packages conforming to the text pattern. Example: `dpkg -l "*nginx*"` isolates web service payloads.
* `dpkg -L <package_name>` : Extract all precise filesystem paths generated by the target package.
* `dpkg -S <pattern>` : Reverse audit. Identify the parent package responsible for generating a specific installed file or path. Example: `dpkg -S /etc/ssh/sshd_config`.
* `dpkg-query -W -f='${Section} ${binary:Package}\n'` : Format the query to extract highly specific data fields.
* `sudo dpkg -i <filename.deb>` : Directly deploy a local raw Debian package binary. Missing dependencies must be resolved manually post-execution using `sudo apt --fix-broken install`.

Package format alternatives and advisories:

- `snap` or `docker` may be preferable when package compatibility or version conflicts prevent a standard `apt` install. Use them only when containment or reproducibility is required:
	- `snap` packages include confinement and automatic updates; review snapd policies before production use.
	- `docker` containers encapsulate full runtime stacks and are best used for isolated services and CI workflows, not as a drop-in replacement for system packages.

Notes:
- Prefer distribution-packaged (`apt`/`dpkg`) software for system-wide tools and daemons; use `snap` or `docker` for reproducible, isolated workloads or where backporting is impractical.

Storage note for snaps:
- `snap` packages are typically mounted as loopback squashfs images under `/snap`. Inspect kernel-recognized loop devices with:
```bash
ls -l /snap
cat /proc/partitions | grep loop
losetup -a
```
- This is normal; snaps use loop devices to present read-only compressed filesystems. When auditing storage or partitions, include loop devices in your checks.

Examples — formatted `dpkg-query` use cases:
```bash
# List installed packages grouped by section (useful for audits)
dpkg-query -W -f='${Section} ${binary:Package}\n' | sort | uniq -c | sort -rn

# Find package names that are exactly four characters long
dpkg-query -W -f='${Section} ${binary:Package}\n' | egrep ' .{4}$'

# List package names and versions (sorted)
dpkg-query -W -f='${Package} ${Version}\n' | sort
```

## Log Auditing (Security Context)
Tracking package operations guarantees visibility over unauthorized modifications.
* `sudo tail -n 100 /var/log/auth.log` : Inspect authentication failures and authorization anomalies during package manipulation steps.
* `cat /var/log/dpkg.log` : Verify the absolute timeline of package additions, upgrades, and removals.
